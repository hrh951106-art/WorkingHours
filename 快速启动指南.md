# 精益工时管理系统 - 快速启动指南

## 🚀 快速开始

### 方式一：本地开发环境启动

#### 1. 启动数据库

**使用Docker（推荐）**:
```bash
docker run -d \
  --name jy-postgres \
  -e POSTGRES_USER=jy_admin \
  -e POSTGRES_PASSWORD=jy_password_2024 \
  -e POSTGRES_DB=jy_working_hours \
  -p 5432:5432 \
  postgres:15-alpine
```

#### 2. 启动后端

```bash
cd backend

# 安装依赖（如未安装）
npm install

# 生成Prisma客户端
npx prisma generate

# 推送数据库结构
npx prisma db push

# 填充初始数据（可选）
npx prisma db seed

# 启动开发服务器
npm run start:dev
```

后端将运行在: **http://localhost:3001**
API文档: **http://localhost:3001/api-docs**

#### 3. 启动前端

```bash
cd frontend

# 安装依赖（如未安装）
npm install

# 启动开发服务器
npm run dev
```

前端将运行在: **http://localhost:5173**

---

### 方式二：Docker Compose 启动

```bash
# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f backend
docker-compose logs -f frontend

# 停止服务
docker-compose down
```

## 📱 功能模块导航

登录系统后，可访问以下模块：

### 1. 工作台
- **路径**: `/dashboard`
- **功能**: 查看系统概况、快速操作入口

### 2. 人事管理
- **组织管理**: `/hr/organizations`
  - 查看/创建/编辑/删除组织
  - 树形结构展示
- **人员管理**: `/hr/employees`
  - 员工列表、搜索、分页
  - 创建/编辑员工信息
  - 查看变更时间轴

### 3. 劳动力账户
- **路径**: `/account`
- **功能**: 账户列表/树形展示、层级配置、主/子账户管理

### 4. 打卡管理
- **设备管理**: `/punch/devices`
  - 设备列表、添加设备
- **打卡记录**: `/punch/records`
  - 记录查询、手动补录
- **异常记录**: `/punch/exceptions`

### 5. 排班管理
- **班次管理**: `/shift/shifts`
  - 班次配置（多时间段）
- **排班管理**: `/shift/schedules`
  - 日历排班、批量排班、调整排班
  - 子账户转移

### 6. 工时计算
- **路径**: `/calculate`
- **功能**: 规则配置、执行计算、查看结果

### 7. 考勤管理
- **路径**: `/attendance`
- **功能**: 个人/部门/账户考勤视图、报表生成

## 🎨 核心功能演示

### 演示1：创建组织和员工

1. 访问 `/hr/organizations`
2. 点击"新建组织"，创建层级结构：
   - 集团 → 公司 → 部门
3. 访问 `/hr/employees`
4. 点击"新建人员"，录入员工信息
5. 保存后可在列表中查看

### 演示2：排班

1. 访问 `/shift/shifts`
2. 点击"新建班次"，配置：
   - 班次名称：正常班
   - 时间段：08:00-12:00, 13:30-17:30
   - 标准工时：8小时
3. 访问 `/shift/schedules`
4. 点击"新建排班"，选择员工和班次
5. 在日历中查看排班情况

### 演示3：工时计算

1. 确保已有排班数据
2. 访问 `/calculate`
3. 选择计算日期
4. 点击"执行计算"
5. 查看计算结果（标准工时、实际工时、加班工时等）

### 演示4：查看考勤报表

1. 访问 `/attendance`
2. 选择日期范围
3. 查看统计数据：
   - 出勤天数
   - 标准工时总计
   - 实际工时总计
   - 加班工时总计
4. 在表格中查看每日明细

## ⚠️ 常见问题

### Q1: 数据库连接失败
**A**: 检查PostgreSQL是否启动，端口5432是否被占用
```bash
# 检查Docker容器状态
docker ps | grep jy-postgres

# 查看数据库日志
docker logs jy-postgres
```

### Q2: Prisma生成失败
**A**: 确保在backend目录下运行命令
```bash
cd backend
npx prisma generate
npx prisma db push
```

### Q3: 前端无法访问后端API
**A**: 检查后端是否启动，端口是否正确
```bash
# 检查后端服务
curl http://localhost:3001/api/auth/profile

# 检查前端代理配置（vite.config.ts）
```

### Q4: 登录失败
**A**: 使用默认账号：
- admin / admin123
- hr_admin / hr123

### Q5: 页面空白或报错
**A**: 清除缓存重新安装
```bash
cd frontend
rm -rf node_modules
npm install
npm run dev
```
