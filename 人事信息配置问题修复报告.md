# 人事信息配置问题修复报告

## 问题概述
用户报告："人事信息配置里面的字段全部没了"

## 问题诊断

### 1. 数据库检查结果
- **EmployeeInfoTab** (人事信息页签表): 0条记录 ❌
- **EmployeeInfoTabField** (人事信息字段表): 0条记录 ❌
- **EmployeeInfoTabGroup** (人事信息分组表): 0条记录 ❌
- **CustomField** (自定义字段表): 0条记录

### 2. 问题原因分析
通过代码分析发现，`getEmployeeInfoConfigs()` API 返回的数据包含两部分：
1. **页签字段**（EmployeeInfoTabField）- 只返回配置了数据源（dataSourceId不为空）的字段
2. **自定义字段**（CustomField）

由于这些表都是空的，所以API返回空数据，前端页面看不到任何字段配置。

## 修复步骤

### 步骤1: 恢复人事信息页签配置
执行脚本：`prisma/seed-employee-info-tabs.ts`

```bash
npx ts-node prisma/seed-employee-info-tabs.ts
```

**结果**：
- ✓ 创建了5个页签：
  1. 基本信息
  2. 工作信息
  3. 学历信息
  4. 工作经历
  5. 家庭信息
- ✓ 创建了6个分组
- ✓ 创建了34个字段

### 步骤2: 初始化数据源
执行脚本：`prisma/init-datasources-for-fields.sql`

```bash
sqlite3 prisma/dev.db < prisma/init-datasources-for-fields.sql
```

**结果**：
- ✓ 创建了11个新的数据源：
  - 性别 (GENDER)
  - 婚姻状况 (MARITAL_STATUS)
  - 政治面貌 (POLITICAL_STATUS)
  - 紧急联系人关系 (EMERGENCY_CONTACT_RELATION)
  - 工作关系 (EMPLOYMENT_RELATION)
  - 入职类型 (ENTRY_TYPE)
  - 员工类型 (EMPLOYEE_TYPE)
  - 职级 (JOB_LEVEL)
  - 岗位 (JOB_POST)
  - 成本中心 (COST_CENTER)
  - 职务 (POSITION_TITLE)

### 步骤3: 关联字段与数据源
执行脚本：`prisma/link-fields-to-datasources.sql`

```bash
sqlite3 prisma/dev.db < prisma/link-fields-to-datasources.sql
```

**结果**：
- ✓ 关联了6个下拉字段到对应的数据源：
  1. gender (性别) → GENDER 数据源
  2. maritalStatus (婚姻状况) → MARITAL_STATUS 数据源
  3. politicalStatus (政治面貌) → POLITICAL_STATUS 数据源
  4. employeeType (员工类型) → EMPLOYEE_TYPE 数据源
  5. jobLevel (职级) → JOB_LEVEL 数据源
  6. status (员工状态) → WORK_STATUS 数据源

## 后续建议

### 1. 添加更多数据源关联
目前只有6个字段关联了数据源，可以根据需要为其他SELECT类型的字段添加数据源关联：

```sql
-- 示例：为学历字段添加数据源
UPDATE EmployeeInfoTabField SET dataSourceId = 2 WHERE fieldCode = 'education';

-- 示例：为岗位字段添加数据源
UPDATE EmployeeInfoTabField SET dataSourceId = 14 WHERE fieldCode = 'position';
```

### 2. 数据备份建议
为了避免数据再次丢失，建议：
1. 定期备份数据库
2. 在执行数据库迁移或schema更新前，先备份重要数据
3. 使用版本控制管理初始化脚本

### 3. 添加自定义字段
如果需要添加更多字段，可以：
1. 通过管理界面创建自定义字段（CustomField）
2. 或直接在数据库中添加，然后关联数据源

```sql
-- 示例：添加自定义字段并关联数据源
INSERT INTO CustomField (code, name, type, dataSourceId, status, sort)
VALUES ('work_years', '工作年限', 'SELECT', 15, 'ACTIVE', 1);
```
