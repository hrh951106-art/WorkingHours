# 生产环境部署检查清单

本文档提供了精益工时管理系统部署到生产环境前的检查清单。

## 部署前准备

### 服务器配置
- [ ] CPU: 4核心或以上
- [ ] 内存: 8GB或以上
- [ ] 磁盘: 50GB+ 可用空间 (SSD推荐)
- [ ] 操作系统: Ubuntu 20.04+, CentOS 7+, Debian 10+
- [ ] Node.js 18+ 或 20.x 已安装
- [ ] npm 9+ 或 10.x 已安装
- [ ] PostgreSQL 13+ 或 MySQL 8.0+ 已安装
- [ ] Nginx 1.18+ 已安装

### 安全配置
- [ ] 防火墙已配置 (UFW/iptables)
- [ ] SSH密钥认证已启用
- [ ] root登录已禁用
- [ ] 默认SSH端口已修改 (可选)
- [ ] fail2ban已安装并配置

### 网络配置
- [ ] 域名已解析到服务器IP
- [ ] DNS记录已配置 (A记录/CNAME记录)
- [ ] SSL证书已准备或已配置Let's Encrypt

## 重要提示

1. **安全第一**: 确保所有默认密码都已修改，所有密钥都已更新
2. **备份优先**: 在做任何变更前，确保数据已备份
3. **逐步推进**: 建议先在测试环境验证，再部署到生产环境
4. **监控先行**: 确保监控和告警已配置，才能及时发现问题
5. **文档齐全**: 确保所有配置和流程都有文档记录

## 常见检查失败项

- JWT_SECRET使用默认值或弱密码 ❌
- 数据库密码过于简单 ❌
- 没有配置备份 ❌
- 没有配置日志轮转 ❌
- SSL证书未配置或过期 ❌
- 防火墙未配置 ❌
- 默认账号密码未修改 ❌

## 快速检查命令

```bash
# 检查服务状态
pm2 status
systemctl status nginx
systemctl status postgresql

# 检查端口监听
sudo netstat -tlnp | grep -E ':(80|443|3001|5432)'

# 检查磁盘使用
df -h

# 检查内存使用
free -m

# 检查日志
tail -f ~/.pm2/logs/jy-backend-error.log
tail -f /var/log/nginx/jy-frontend-error.log

# 测试API
curl -I https://your-domain.com/api/system/dashboard/stats

# 检查SSL证书
echo | openssl s_client -servername your-domain.com -connect your-domain.com:443 2>/dev/null | openssl x509 -noout -dates
```
