## 📊 六、人事信息页签配置 (EmployeeInfoTab)

### 6.1 查询SQL

```sql
-- 查询所有人事信息页签配置
SELECT
    t.id,
    t.code,
    t.name,
    t.description,
    t.isSystem,
    t.sort,
    t.status,
    COUNT(DISTINCT g.id) AS groupCount,
    COUNT(DISTINCT f.id) AS fieldCount
FROM EmployeeInfoTab t
LEFT JOIN EmployeeInfoTabGroup g ON g.tabId = t.id
LEFT JOIN EmployeeInfoTabField f ON f.tabId = t.id
WHERE t.code IN ('basic_info', 'work_info', 'education_info', 'work_experience', 'family_info')
GROUP BY t.id
ORDER BY t.sort;

-- 查询页签分组和字段详情
SELECT
    t.code AS tabCode,
    t.name AS tabName,
    g.code AS groupCode,
    g.name AS groupName,
    f.fieldCode,
    f.fieldName,
    f.fieldType,
    f.isRequired,
    f.isHidden,
    f.sort
FROM EmployeeInfoTab t
LEFT JOIN EmployeeInfoTabGroup g ON g.tabId = t.id
LEFT JOIN EmployeeInfoTabField f ON f.tabId = t.id AND f.groupId = g.id
WHERE t.code IN ('basic_info', 'work_info', 'education_info', 'work_experience', 'family_info')
ORDER BY t.sort, g.sort, f.sort;
```

### 6.2 实际配置结果

#### 基本信息 (basic_info)
- **分组数**: 2个
- **字段数**: 21个
- **分组1**: 个人信息 (personal_info) - 16个字段
  - 姓名、性别、身份证号、出生日期、年龄、民族、婚姻状况、政治面貌、籍贯、户口所在地、现居住地址、照片、手机号码、邮箱、设备类型、人员类型
- **分组2**: 紧急联系人 (emergency_contact) - 5个字段
  - 紧急联系人、紧急联系电话、紧急联系人关系、家庭住址、家庭电话

#### 工作信息 (work_info)
- **分组数**: 2个
- **字段数**: 17个
- **分组1**: 职位信息 (position_info) - 6个字段
  - 所属组织、职位、职级、员工类型、工作地点、办公地址
- **分组2**: 入职信息 (entry_info) - 11个字段
  - 工号、入职日期、受雇日期、试用期开始、试用期结束、试用期月数、转正日期、在职状态、离职日期、离职原因、工作年限

#### 学历信息 (education_info)
- **分组数**: 1个
- **字段数**: 7个
- **分组1**: 最高学历 (highest_education) - 7个字段
  - 学历层次、学历类型、毕业院校、专业、毕业时间、学位证书号、毕业证书号

#### 工作经历 (work_experience)

  - 公司名称、职位、开始时间、结束时间、离职时薪资、离职原因、工作描述

## 🔍 七、综合查询示例

### 7.1 查询员工完整信息

```sql
-- 查询员工基本信息、学历、工作经历、家庭成员
SELECT
    e.id AS employeeId,
    e.employeeNo,
    e.name AS employeeName,
    e.gender,
    e.phone,
    e.email,
    e.entryDate,
    e.status,
    o.name AS organizationName,

    -- 学历信息
    (SELECT COUNT(*) FROM EmployeeEducation WHERE employeeId = e.id) AS educationCount,
    (SELECT school FROM EmployeeEducation WHERE employeeId = e.id AND isHighest = 1 LIMIT 1) AS highestSchool,

    -- 工作经历
    (SELECT COUNT(*) FROM EmployeeWorkExperience WHERE employeeId = e.id) AS workExpCount,

    -- 家庭成员
    (SELECT COUNT(*) FROM EmployeeFamilyMember WHERE employeeId = e.id) AS familyCount

FROM Employee e
LEFT JOIN Organization o ON o.id = e.orgId
WHERE e.status = 'ACTIVE'
ORDER BY e.entryDate DESC;
```

### 7.2 查询员工完整档案（包含所有子表）

```sql
-- 主查询
WITH EmployeeBasic AS (
    SELECT
        e.id,
        e.employeeNo,
        e.name,
        e.gender,
        e.phone,
        e.email,
        e.orgId,
        o.name AS orgName,
        e.entryDate,
        e.status
    FROM Employee e
    LEFT JOIN Organization o ON o.id = e.orgId
),
EmployeeEducations AS (
    SELECT
        employeeId,
        GROUP_CONCAT(school || ':' || major, '; ') AS educations
    FROM EmployeeEducation
    GROUP BY employeeId
),
EmployeeWorkExps AS (
    SELECT
        employeeId,
        GROUP_CONCAT(company || ':' || position, '; ') AS workExps
    FROM EmployeeWorkExperience
    GROUP BY employeeId
),
EmployeeFamilies AS (
    SELECT
        employeeId,
        GROUP_CONCAT(relationship || ':' || name, '; ') AS families
    FROM EmployeeFamilyMember
    GROUP BY employeeId
)
SELECT
    eb.*,
    ee.educations,
    ew.workExps,
    ef.families
FROM EmployeeBasic eb
LEFT JOIN EmployeeEducations ee ON ee.employeeId = eb.id
LEFT JOIN EmployeeWorkExps ew ON ew.employeeId = eb.id
LEFT JOIN EmployeeFamilies ef ON ef.employeeId = eb.id
ORDER BY eb.entryDate DESC;
```

## 📝 八、数据完整性检查SQL

### 8.1 检查孤立记录

```sql
-- 检查学历表中没有对应员工的记录
SELECT COUNT(*) AS orphanEducations
FROM EmployeeEducation ee
WHERE NOT EXISTS (SELECT 1 FROM Employee e WHERE e.id = ee.employeeId);

-- 检查工作经历表中没有对应员工的记录
SELECT COUNT(*) AS orphanWorkExps
FROM EmployeeWorkExperience ew
WHERE NOT EXISTS (SELECT 1 FROM Employee e WHERE e.id = ew.employeeId);

-- 检查家庭成员表中没有对应员工的记录
SELECT COUNT(*) AS orphanFamilies
FROM EmployeeFamilyMember ef
WHERE NOT EXISTS (SELECT 1 FROM Employee e WHERE e.id = ef.employeeId);
```

### 8.2 检查数据一致性

```sql
-- 检查员工orgId是否有效
SELECT e.id, e.employeeNo, e.name, e.orgId
FROM Employee e
WHERE e.orgId IS NOT NULL
  AND NOT EXISTS (SELECT 1 FROM Organization o WHERE o.id = e.orgId);

-- 检查WorkInfoHistory中的orgId是否有效
SELECT w.id, w.employeeId, w.orgId
FROM WorkInfoHistory w
WHERE w.orgId IS NOT NULL
  AND NOT EXISTS (SELECT 1 FROM Organization o WHERE o.id = w.orgId);
```

## 🎯 总结

### 表结构关系图

```
Employee (员工主表)
├── EmployeeEducation (学历信息) - 一对多
├── EmployeeWorkExperience (工作经历) - 一对多
├── EmployeeFamilyMember (家庭成员) - 一对多
├── WorkInfoHistory (工作信息变更历史) - 一对多
├── EmployeeChangeLog (员工变更日志) - 一对多
└── Organization (组织架构) - 多对一

EmployeeInfoTab (人事信息页签配置)
├── EmployeeInfoTabGroup (页签分组) - 一对多
└── EmployeeInfoTabField (页签字段) - 一对多
```

### 关键特点

1. **Employee表**包含员工的基本信息和工作信息
2. **学历、工作经历、家庭成员**使用独立的子表存储，支持多条记录
3. **WorkInfoHistory表**记录工作信息的历史变更
4. **EmployeeInfoTab系列表**提供灵活的人事信息配置系统
5. **customFields字段**使用JSON格式存储自定义字段数据
