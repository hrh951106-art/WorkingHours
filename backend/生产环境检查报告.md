### ⚠️ 发现的问题

#### 问题1: 重复的人事信息页签

**严重程度**: 中等

**描述**: 数据库中同时存在大写代码（BASIC_INFO）和小写代码（basic_info）的页签，导致前端显示重复或混乱。

**影响**:
- 前端可能显示两个相同的页签
- 其中一个页签可能没有分组和字段
- 用户不知道应该填写哪个页签

**状态**:
- ✓ 开发环境：已发现
- ⚠️ 生产环境：需要检查

**解决方案**: 运行 `scripts/cleanup-duplicate-tabs.sql` 清理重复页签

#### 问题2: gender和age字段可能缺失

**严重程度**: 高

**描述**: Employee表中存在gender和age字段，但人事信息页签配置（EmployeeInfoTabField）中可能缺少这两个字段的配置。

**影响**:
- 前端查询页签字段配置时找不到gender和age
- 导致接口报错："字段不存在"
- 员工信息无法正常显示或编辑

**状态**:
- ✓ 开发环境：已确认字段存在
- ⚠️ 生产环境：需要验证

**解决方案**: 运行 `scripts/fix-employee-fields.sql` 添加缺失字段

## 🚨 生产环境检查清单

### 第一步：运行完整检查脚本

```bash
# PostgreSQL生产环境
psql -U jy_admin -d jy_production -f scripts/check-all-employee-tables.sql
```

**预期输出**:
- 所有8个表都应该存在
- 5个系统页签配置应该完整
- 每个页签应该有正确的分组和字段数量
- gender和age字段应该存在

### 第二步：检查重复页签

```sql
-- 检查是否有重复的页签代码
SELECT code, name, COUNT(*) AS count
FROM "EmployeeInfoTab"
GROUP BY code, name
HAVING COUNT(*) > 1;
```

**如果有重复**:
- 运行清理脚本删除大写代码的重复页签

### 第三步：检查关键字段

```sql
-- 检查gender和age字段是否存在
SELECT
    t.code AS tabCode,
    f.fieldCode,
    f.fieldName,
    f.isRequired
FROM "EmployeeInfoTab" t
INNER JOIN "EmployeeInfoTabField" f ON f."tabId" = t.id
WHERE t.code = 'basic_info'
  AND f.fieldCode IN ('gender', 'age');
```

**如果缺失**:
- 运行修复脚本添加字段

### 第四步：数据完整性验证

```sql
-- 检查孤立记录
SELECT 'EmployeeEducation孤立记录' AS checkItem, COUNT(*) AS count
FROM "EmployeeEducation" e
WHERE NOT EXISTS (SELECT 1 FROM "Employee" emp WHERE emp.id = e."employeeId")
UNION ALL
SELECT 'EmployeeWorkExperience孤立记录', COUNT(*)
FROM "EmployeeWorkExperience" e
WHERE NOT EXISTS (SELECT 1 FROM "Employee" emp WHERE emp.id = e."employeeId")
UNION ALL
SELECT 'EmployeeFamilyMember孤立记录', COUNT(*)
FROM "EmployeeFamilyMember" e
WHERE NOT EXISTS (SELECT 1 FROM "Employee" emp WHERE emp.id = e."employeeId");
```


**缺点**:


**优点**:

- 只清理配置重复

**缺点**:

**优点**:
- 精准修复字段问题
- 不影响其他配置

**缺点**:
- 不解决重复页签问题

### 方案4: 综合修复（推荐）
