### 问题1: 保存失败 - gender字段必填验证

**原因**:
- Employee表中gender字段是必填的（NOT NULL）
- 创建员工时，如果前端没有提供gender或值为空，就会报错
- 原代码没有对gender字段进行验证

**修复方案**:
修改了 `src/modules/hr/hr.service.ts` 的第275-300行，添加了gender字段的验证逻辑：

1. **必填验证**: 如果gender为空，抛出明确的错误提示
2. **值合法性验证**: 检查gender值是否在有效范围内
3. **值格式统一**: 将中文"男"/"女"转换为英文"male"/"female"

**修复代码**:
```typescript
// gender字段必填处理
if (key === 'gender') {
  if (!value || value === '' || value === null || value === undefined) {
    throw new BadRequestException('性别为必填字段，请选择性别');
  }
  // 验证gender值是否合法
  const validGenders = ['male', 'female', '男', '女'];
  if (!validGenders.includes(value)) {
    throw new BadRequestException(`性别值无效: ${value}，有效值为: male, female, 男, 女`);
  }
  // 统一转换为英文值
  if (value === '男') {
    finalValue = 'male';
  } else if (value === '女') {
    finalValue = 'female';
  }
}
```

