> **更新日期**: 2026-04-01

## 📋 更新概述

本次更新主要修复人事管理模块中系统字段无法选择数据源、字段类型错误、前端验证逻辑不完整等问题。

### 核心问题

1. ❌ **系统内置字段缺失CustomField记录** - gender、nation等11个系统字段没有CustomField配置
2. ❌ **字段类型不统一** - 数据库使用SELECT，代码和前端期望SELECT_SINGLE
3. ❌ **前端过滤逻辑不完整** - 只过滤SYSTEM类型，导致SELECT_SINGLE字段被忽略
4. ❌ **后端硬编码验证** - gender字段值被限制为male/female/男/女
5. ❌ **数据源重复** - 部分数据源存在重复记录

### 修复内容

- ✅ 为所有系统内置字段创建CustomField记录并关联数据源
- ✅ 统一所有字段类型为SELECT_SINGLE
- ✅ 修复前端4处fieldType判断逻辑，包含SYSTEM/SELECT_SINGLE/SELECT_MULTI
- ✅ 去掉后端gender字段的硬编码值验证，改为只验证非空
- ✅ 删除重复的数据源记录
- ✅ 前后端生产环境构建

if (key === 'gender') {
  if (!value || value === '' || value === null || value === undefined) {
    throw new BadRequestException('性别为必填字段，请选择性别');
  }
  // 验证gender值是否合法
  const validGenders = ['male', 'female', '男', '女'];
  if (!validGenders.includes(value)) {
    throw new BadRequestException(`性别值无效: ${value}，有效值为: male, female, 男, 女`);

  // 统一转换为英文值

## 🔍 常见问题

### Q1: 执行SQL时报错"relation does not exist"

**原因**: 表名大小写或引号问题

**解决**: PostgreSQL表名默认转小写，确保使用双引号保持原名：
```sql
SELECT * FROM "EmployeeInfoTabField";  -- 正确
SELECT * FROM EmployeeInfoTabField;   -- 错误
```

---

### Q2: 执行SQL后字段仍然无法选择

**排查步骤**:
1. 检查CustomField记录是否存在
2. 检查dataSourceId是否正确
3. 检查DataSourceOption是否有选项
4. 清除浏览器缓存并刷新页面
5. 检查前端控制台是否有错误

---

### Q3: 保存时仍然提示"性别值无效"

**原因**: 后端代码未更新，仍在运行旧版本

**解决**:
1. 确认后端服务已重启
2. 检查运行的代码版本是否为最新
3. 查看后端日志确认没有启动失败
4. 清除后端编译缓存：`rm -rf dist && npm run build`

---

### Q4: 前端页面空白或报错

**原因**: 前端构建产物未正确部署

**解决**:
1. 检查nginx或静态文件服务配置
2. 确认dist目录文件完整
3. 检查浏览器控制台具体错误信息
4. 清除浏览器缓存
