# 字段类型不匹配问题修复报告

> **修复时间**: 2025-04-01 10:39
> **问题核心**: 字段类型不匹配导致前端无法渲染

---

## 🔍 问题诊断

### 用户反馈的问题

基本信息页签：
- 配置显示：21个字段
- 新增页面显示：16个字段
- **缺失字段**：5个
  1. gender（性别）
  2. nation（民族）
  3. politicalStatus（政治面貌）
  4. maritalStatus（婚姻状况）
  5. birthDate（出生日期）

### 根本原因分析

**字段类型不匹配**：

| 表/位置 | 字段类型 | 说明 |
|---------|---------|------|
| CustomField.type | `SELECT_SINGLE` | ✅ 正确 |
| EmployeeInfoTabField.fieldType | `SELECT` | ❌ 错误 |
| 前端代码处理 | `SELECT_SINGLE` | ✅ 只处理这个类型 |

**问题链路**:
1. CustomField表使用 `SELECT_SINGLE` 类型
2. EmployeeInfoTabField表使用 `SELECT` 类型（修复SQL中设置的）
3. 前端代码只处理 `SELECT_SINGLE` 和 `SELECT_MULTI` 类型
4. 类型不匹配 → 前端跳过这些字段 → 字段不显示

---

## ✅ 修复方案

### 修复步骤

**执行SQL**: `backend/scripts/fix-field-type-mismatch.sql`

```sql
UPDATE EmployeeInfoTabField
SET fieldType = 'SELECT_SINGLE'
WHERE fieldType = 'SELECT';
```
