## ⚠️ 问题描述

**用户报告**:
- 生产环境在新增人员时，性别、民族等字段不显示
- 人事信息配置中是配置显示的
- 本地环境正常，生产环境异常

---

## 🔍 根本原因分析

### 数据层面

| 检查项 | 本地环境 | 生产环境 | 影响 |
|-------|---------|---------|------|
| **EmployeeInfoTabField.fieldType** | SYSTEM | SELECT_SINGLE | 🔴 关键 |
| **CustomField记录** | 无 | 无/部分 | 🟡 中等 |
| **DataSource数据** | 有 | 有 | 🟢 正常 |
| **DataSourceOption数据** | 有 | 有 | 🟢 正常 |

### 代码层面

```typescript
// 前端过滤逻辑（旧代码）
const systemFields = group.fields?.filter((f: any) =>
  f.fieldType === 'SYSTEM' && !f.isHidden  // ❌ 只识别SYSTEM
) || [];
```

### 问题链条

```
生产环境数据库:
  └─ fieldType = 'SELECT_SINGLE'
      ↓
前端代码:
  └─ 只识别 fieldType === 'SYSTEM'
      ↓
过滤结果:
  └─ SELECT_SINGLE字段被过滤掉
      ↓
最终表现:
  └─ 性别等字段不显示 ❌
```

- → 执行方案B完整修复

**情况C：需要紧急恢复**
- → 先执行方案A快速修复
- → 后续执行方案B完整修复

**方案B（完整）**: 30分钟

### 第四步：验证（5分钟）

```sql
-- 验证CustomField记录
SELECT COUNT(*) FROM "CustomField"
WHERE code IN ('gender', 'nation', 'maritalStatus');
-- 期望: 3

-- 验证字段类型
SELECT fieldCode, fieldType FROM "EmployeeInfoTabField"
WHERE fieldCode IN ('gender', 'nation', 'maritalStatus');
-- 期望: fieldType = 'SELECT_SINGLE'

-- 验证数据源
SELECT cf.code, ds.code AS dataSource, COUNT(dso.id) AS options
FROM "CustomField" cf
LEFT JOIN "DataSource" ds ON ds.id = cf."dataSourceId"
LEFT JOIN "DataSourceOption" dso ON dso."dataSourceId" = ds.id
WHERE cf.code IN ('gender', 'nation', 'maritalStatus')
GROUP BY cf.code, ds.code;
-- 期望: options > 0
```
