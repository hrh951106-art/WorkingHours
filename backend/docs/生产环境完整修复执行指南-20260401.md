## 🎯 完整方案说明

### 为什么方案A不行？

**方案A（快速修复）**:
```sql
UPDATE "EmployeeInfoTabField" SET "fieldType" = 'SYSTEM'
```

**结果**:
- ✅ 字段显示了
- ❌ 但类型是SYSTEM，不会关联CustomField
- ❌ 无法使用数据源下拉框

**根本原因**:
```
SYSTEM类型字段 → 直接渲染为输入框 → 无数据源
SELECT_SINGLE类型 → 关联CustomField → 获取DataSource → 显示下拉框
```

SELECT
    cf.code,
    cf.name,
    ds.code AS dataSource,
    COUNT(dso.id) AS options,

GROUP BY cf.code, cf.name, ds.code, cf.id;

## ⚠️ 常见问题

### Q1: SQL执行时提示"数据源不存在"

**原因**: DataSource表中缺少对应的数据源记录

**解决**: 先创建数据源和选项

```sql
-- 示例：创建gender数据源
INSERT INTO "DataSource" (code, name, type, "isSystem", status, createdAt, updatedAt)
VALUES ('gender', '性别', 'BUILTIN', true, 'ACTIVE', CURRENT_TIMESTAMP, CURRENT_TIMESTAMP);

-- 获取刚创建的数据源ID
SELECT id, code FROM "DataSource" WHERE code = 'gender';

-- 添加选项（假设数据源ID为1）
INSERT INTO "DataSourceOption" ("dataSourceId", value, label, sort, createdAt, updatedAt)
VALUES
  (1, '01', '男', 1, CURRENT_TIMESTAMP, CURRENT_TIMESTAMP),
  (1, '02', '女', 2, CURRENT_TIMESTAMP, CURRENT_TIMESTAMP);
```

### Q2: 字段显示了但不是下拉框

**原因**: 前端代码未更新或未部署

**解决**:
1. 确认前端代码已更新（包含SELECT_SINGLE）
2. 确认前端已重新构建
3. 确认新构建的代码已部署
4. 清除浏览器缓存

### Q3: 字段显示但下拉框无选项

**原因**: CustomField未关联DataSource或DataSource无选项

**解决**:
```sql
-- 检查关联
SELECT
    cf.code,
    cf."dataSourceId",
    ds.code AS dataSourceCode,
    COUNT(dso.id) AS optionCount
FROM "CustomField" cf
LEFT JOIN "DataSource" ds ON ds.id = cf."dataSourceId"
LEFT JOIN "DataSourceOption" dso ON dso."dataSourceId" = ds.id
WHERE cf.code = 'gender'
GROUP BY cf.code, cf."dataSourceId", ds.code;
```

### Q4: 保存时提示"性别值无效"

**原因**: 后端代码未更新

**解决**:
1. 确认后端代码已去掉gender硬编码验证
2. 重新构建并部署后端
3. 重启后端服务
