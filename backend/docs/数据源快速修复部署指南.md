# 系统内置字段数据源快速修复部署指南

> **方案**: SQL视图 + 新API端点
> **时间**: 5分钟立即生效
> **风险**: 极低，不影响现有功能

---

## 📋 问题说明

### 问题描述
生产环境中，系统内置的下拉字段（性别、民族、婚姻状况等）无法从查找项中获取数据源选项，导致前端下拉框为空。

### 根本原因
后端代码 `src/modules/hr/employee-info-tab.service.ts` 中的 `enrichFieldWithType` 方法只处理 `fieldType === 'CUSTOM'` 的字段，而系统内置字段的 `fieldType` 是 `'SYSTEM'`，因此无法获取数据源。

### 修复方案
创建PostgreSQL视图和函数，直接关联 `CustomField` 和 `DataSource` 表，绕过原有代码限制，提供包含完整数据源的新API端点。


**解决**: 确保所有基础表都有数据

### 问题2：字段没有dataSource

**检查**:
```sql
-- 检查CustomField是否有对应记录
SELECT * FROM "CustomField"
WHERE code IN ('gender', 'nation', 'maritalStatus');

-- 检查CustomField是否关联了DataSource
SELECT
    cf.code,
    cf.name,
    d.code AS datasource_code
FROM "CustomField" cf
LEFT JOIN "DataSource" d ON d.id = cf."dataSourceId"
WHERE cf.code IN ('gender', 'nation', 'maritalStatus');
```

**解决**:
```sql
-- 运行修复脚本创建CustomField记录
\i scripts/fix-datasource-fields.sql
```

### 问题3：选项为空数组

**检查**:
```sql
-- 检查DataSourceOption表
SELECT
    d.code AS datasource_code,
    COUNT(o.id) AS option_count
FROM "DataSource" d
LEFT JOIN "DataSourceOption" o ON o."dataSourceId" = d.id AND o."isActive" = true
WHERE d.code IN ('GENDER', 'NATION', 'MARITAL_STATUS')
GROUP BY d.code, d.name;
```

**解决**:
```sql
-- 如果GENDER数据源没有选项，插入选项
INSERT INTO "DataSourceOption" ("dataSourceId", label, value, sort, "isActive", "createdAt", "updatedAt")
SELECT
    (SELECT id FROM "DataSource" WHERE code = 'GENDER'),
    '男',
    'MALE',
    1,
    true,
    NOW(),
    NOW()
UNION ALL
SELECT
    (SELECT id FROM "DataSource" WHERE code = 'GENDER'),
    '女',
    'FEMALE',
    2,
    true,
    NOW(),
    NOW();
```

## 📈 性能优化

如果数据量较大，可以考虑以下优化：

### 优化1：添加索引

```sql
-- 为CustomField添加code索引
CREATE INDEX IF NOT EXISTS idx_custom_field_code
ON "CustomField"(code);

-- 为DataSource添加code索引
CREATE INDEX IF NOT EXISTS idx_datasource_code
ON "DataSource"(code);

-- 为DataSourceOption添加复合索引
CREATE INDEX IF NOT EXISTS idx_datasource_option_datasource_active
ON "DataSourceOption"("dataSourceId", "isActive");
```

### 优化2：缓存结果

在服务中添加缓存：

```typescript
import { Cache } from 'cache-manager';

@Injectable()
export class EmployeeInfoTabQuickFixService {
  constructor(
    private readonly prisma: PrismaService,
    @Inject(CACHE_MANAGER) private cacheManager: Cache,
  ) {}

  async getTabsWithDataSource() {
    const cacheKey = 'employee_info_tabs_with_datasource';

    // 尝试从缓存获取
    let result = await this.cacheManager.get(cacheKey);
    if (result) {
      return result;
    }

    // 查询数据库
    result = await this.prisma.$queryRaw`
      SELECT * FROM get_employee_info_tabs_display()
      ORDER BY code
    `;

    // 缓存1小时
    await this.cacheManager.set(cacheKey, result, { ttl: 3600 });

    return result;
  }
}
```
