# 人事信息配置页签缺失数据问题修复指南

> **问题**: 教育信息、工作经历、家庭信息页签没有分组和字段
> **日期**: 2026-04-01
> **影响**: 页面无法渲染、无法配置、提交时遇到限制

---

## 🔍 问题分析

### 问题现象

1. **人事信息配置页面**
   - 教育信息页签：点击后显示空白或错误
   - 工作经历页签：点击后显示空白或错误
   - 家庭信息页签：点击后显示空白或错误

2. **新增人员页面**
   - 三个页签无法使用
   - 无法添加教育经历、工作经历、家庭成员信息
   - 提交时遇到验证限制

### 根本原因

**数据库缺失配置数据**：

```
应该有的数据结构：
EmployeeInfoTab (页签表)
  ├─ education_info (学历信息页签)
  │   └─ EmployeeInfoTabGroup (分组)
  │       └─ EmployeeInfoTabField (字段 x 7)
  ├─ work_experience (工作经历页签)
  │   └─ EmployeeInfoTabGroup (分组)
  │       └─ EmployeeInfoTabField (字段 x 7)
  └─ family_info (家庭信息页签)
      └─ EmployeeInfoTabGroup (分组)
          └─ EmployeeInfoTabField (字段 x 6)

生产环境实际：
EmployeeInfoTab
  ├─ education_info (页签存在) ✓
  │   └─ EmployeeInfoTabGroup (缺失) ✗
  ├─ work_experience (页签存在) ✓
  │   └─ EmployeeInfoTabGroup (缺失) ✗
  └─ family_info (页签存在) ✓
      └─ EmployeeInfoTabGroup (缺失) ✗
```

### 为什么会这样？

可能的原因：
1. **数据库迁移脚本不完整** - 创建了页签但没有创建分组和字段
2. **手动清理数据** - 误删了分组和字段数据
3. **同步失败** - 从测试环境同步到生产环境时数据不完整

### 第四步：验证修复

执行以下SQL验证：

```sql
-- 快速验证
SELECT
    tabCode,
    tabName,
    groupCount,
    fieldCount,
    status
FROM (
    SELECT
        t.code AS tabCode,
        t.name AS tabName,
        COUNT(DISTINCT g.id) AS groupCount,
        COUNT(f.id) AS fieldCount,
        CASE
            WHEN COUNT(DISTINCT g.id) = 0 THEN '✗ 无分组'
            WHEN COUNT(f.id) = 0 THEN '✗ 无字段'
            ELSE '✓ 完整'
        END AS status
    FROM "EmployeeInfoTab" t
    LEFT JOIN "EmployeeInfoTabGroup" g ON g."tabId" = t.id
    LEFT JOIN "EmployeeInfoTabField" f ON f."groupId" = g.id
    WHERE t.code IN ('education_info', 'work_experience', 'family_info')
    GROUP BY t.code, t.name
) t
ORDER BY tabCode;
```

## 🔑 关键说明

### 为什么分组状态是INACTIVE？

**工作经历**和**家庭信息**的分组状态是`INACTIVE`，这是**正常的设计**：

```
ACTIVE状态：基本信息、工作信息等单值字段
  - 在新增人员表单中直接显示
  - 每个字段输入一次

INACTIVE状态：教育经历、工作经历、家庭成员等数组字段
  - 不在表单中直接显示
  - 通过动态添加的方式（点击"添加"按钮）
  - 可以添加多条记录
```

### API返回结构

修复前：
```json
{
  "code": "education_info",
  "name": "学历信息",
  "groups": [],  // 空数组或null
  "fields": []
```

修复后：

## 📞 常见问题

### Q1: 执行SQL后页面还是显示空白

**解决**:
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 硬刷新页面（Ctrl+Shift+R）
3. 检查前端控制台是否有错误
4. 重启前端服务

### Q2: 字段显示了但无法保存配置

**原因**: 可能是前端权限问题或API问题

**解决**:
1. 检查用户权限
2. 查看API响应（F12 Network）
3. 检查后端日志

### Q3: 新增人员时这三个页签还是无法使用

**原因**: 前端代码可能对空数组有特殊处理

**解决**:
1. 确认API返回groups不为空
2. 检查前端渲染逻辑
3. 可能需要重启前端服务
