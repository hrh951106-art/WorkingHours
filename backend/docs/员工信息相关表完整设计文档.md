## 🗂️ 表关系概览

### ER图

```
Employee (员工主表)
├── EmployeeEducation (学历信息) - 一对多
├── EmployeeWorkExperience (工作经历) - 一对多
├── EmployeeFamilyMember (家庭成员) - 一对多
├── WorkInfoHistory (工作信息历史) - 一对多
│
└── EmployeeInfoTab (人事信息页签配置) - 系统配置
    ├── EmployeeInfoTabGroup (页签分组) - 一对多
    │   └── EmployeeInfoTabField (页签字段) - 一对多
    └── EmployeeInfoTabField (页签字段) - 一对多 (可直接关联页签)
```

### 数据流向

```
Employee表
    ↓
EmployeeInfoTab (配置)
    ↓
EmployeeInfoTabGroup (分组)
    ↓
EmployeeInfoTabField (字段)
    ↓
前端展示和编辑界面
```

## 1️⃣ EmployeeEducation 表设计

### 表说明

存储员工的学历教育信息，支持多条记录（最高学历、第二学历等）。

### 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| **id** | `Int` | ✓ | AUTOINCREMENT | 主键ID |
| **employeeId** | `Int` | ✓ | - | 员工ID，外键关联Employee表 |
| **school** | `String` | ✓ | - | 毕业院校名称 |
| **major** | `String` | ✓ | - | 专业名称 |
| **degree** | `String` | ✓ | - | 学位（学士/硕士/博士） |
| **educationLevel** | `String` | ✓ | - | 学历层次（本科/研究生/博士） |
| **startDate** | `DateTime` | ✓ | - | 入学日期 |
| **endDate** | `DateTime?` | ✗ | NULL | 毕业日期或预计毕业日期 |
| **isHighest** | `Boolean` | ✓ | false | 是否最高学历 |
| **description** | `String?` | ✗ | NULL | 描述或备注 |
| **degreeNo** | `String?` | ✗ | NULL | 学位证书编号 |
| **diplomaNo** | `String?` | ✗ | NULL | 毕业证书编号 |
| **educationType** | `String?` | ✗ | NULL | 学历类型（全日制/在职/函授等） |
| **graduationDate** | `DateTime?` | ✗ | NULL | 毕业时间（具体日期） |
| **createdAt** | `DateTime` | ✓ | now() | 创建时间 |
| **updatedAt** | `DateTime` | ✓ | now() | 更新时间 |

### 索引设计

```sql
-- 单列索引
CREATE INDEX idx_education_employee ON "EmployeeEducation"("employeeId");

-- 复合索引（查询最高学历）
CREATE INDEX idx_education_highest ON "EmployeeEducation"("employeeId", "isHighest");
```

### 外键约束

```sql
FOREIGN KEY ("employeeId") REFERENCES "Employee"(id) ON DELETE CASCADE
```

## 6️⃣ EmployeeInfoTabField 表设计

### 表说明

人事信息页签字段表，定义每个页签或分组下包含的具体字段。

### 表结构

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| **id** | `Int` | ✓ | AUTOINCREMENT | 主键ID |
| **tabId** | `Int` | ✓ | - | 所属页签ID，外键关联EmployeeInfoTab表 |
| **groupId** | `Int?` | ✗ | NULL | 所属分组ID，外键关联EmployeeInfoTabGroup表（可选） |
| **fieldCode** | `String` | ✓ | - | 字段代码（对应Employee表的字段名或自定义字段） |
| **fieldName** | `String` | ✓ | - | 字段显示名称 |
| **fieldType** | `String` | ✓ | - | 字段类型（SYSTEM/CUSTOM/SELECT/TEXT/DATE等） |
| **isRequired** | `Boolean` | ✓ | false | 是否必填 |
| **isHidden** | `Boolean` | ✓ | false | 是否隐藏 |
| **sort** | `Int` | ✓ | 0 | 排序顺序 |
| **createdAt** | `DateTime` | ✓ | now() | 创建时间 |
| **updatedAt** | `DateTime` | ✓ | now() | 更新时间 |

### 约束

```sql
UNIQUE(tabId, fieldCode)
FOREIGN KEY (tabId) REFERENCES "EmployeeInfoTab"(id) ON DELETE CASCADE
FOREIGN KEY (groupId) REFERENCES "EmployeeInfoTabGroup"(id)
```

### fieldType 枚举值

- `SYSTEM`: 系统字段（对应Employee表的字段）
- `CUSTOM`: 自定义字段（存储在Employee.customFields JSON中）
- `SELECT`: 下拉选择（关联DataSource）
- `TEXT`: 文本输入
- `DATE`: 日期选择
- `NUMBER`: 数字输入
- `IMAGE`: 图片上传
- `FILE`: 文件上传

## 🚨 重要字段说明

### ⚠️ gender 和 age 字段

**问题**: 这两个字段经常在生产环境报错缺失

**原因分析**:
1. Employee表中这两个字段**确实存在**
2. 问题在于**人事信息页签配置**中可能缺少这两个字段的配置
3. 前端查询EmployeeInfoTabField时找不到gender和age字段的配置

**解决方案**:
```sql
-- 检查字段是否存在
SELECT * FROM "EmployeeInfoTabField"
WHERE "tabId" = (SELECT id FROM "EmployeeInfoTab" WHERE code = 'basic_info')
  AND fieldCode IN ('gender', 'age');

-- 如果不存在，添加字段
INSERT INTO "EmployeeInfoTabField" ("tabId", "groupId", fieldCode, fieldName, fieldType, isRequired, sort)
VALUES
  (1, 1, 'gender', '性别', 'SYSTEM', true, 3),
  (1, 1, 'age', '年龄', 'SYSTEM', true, 5);
```

## 🔍 生产环境检查脚本

### 完整检查脚本

```bash
# 运行完整检查
psql -U jy_admin -d jy_production -f scripts/check-all-employee-tables.sql
```

### 快速检查命令

```sql
-- 1. 检查所有表是否存在
SELECT table_name, CASE WHEN table_name IS NOT NULL THEN '✓' ELSE '✗' END
FROM (VALUES ('Employee'), ('EmployeeEducation'), ('EmployeeWorkExperience'),
             ('EmployeeFamilyMember'), ('EmployeeInfoTab'), ('EmployeeInfoTabGroup'),
             ('EmployeeInfoTabField'), ('WorkInfoHistory')) AS t(table_name)
LEFT JOIN information_schema.tables t2 ON t2.table_name = t.table_name;

-- 2. 检查系统页签是否完整
SELECT code, name, "isSystem", status
FROM "EmployeeInfoTab"
WHERE code IN ('basic_info', 'work_info', 'education_info', 'work_experience', 'family_info')
ORDER BY sort;

-- 3. 检查gender和age字段
SELECT f.fieldCode, f.fieldName, f.isRequired
FROM "EmployeeInfoTabField" f
INNER JOIN "EmployeeInfoTab" t ON t.id = f."tabId"
WHERE t.code = 'basic_info' AND f.fieldCode IN ('gender', 'age');

-- 4. 检查数据完整性
SELECT

### 问题1: 页签配置缺失

**症状**: 前端看不到某个页签或字段

**诊断**:

-- 检查页签是否存在
SELECT * FROM "EmployeeInfoTab" WHERE code = 'basic_info';

-- 检查页签是否有字段
SELECT COUNT(*) FROM "EmployeeInfoTabField"
WHERE "tabId" = (SELECT id FROM "EmployeeInfoTab" WHERE code = 'basic_info');
```

**解决**:
```bash
# 运行完整初始化脚本
psql -U jy_admin -d jy_production -f prisma/migrations_postgres/20250331_init/init_production.sql
```

### 问题2: 有重复的页签

**症状**: 前端显示两个相同的页签

**诊断**:
```sql
SELECT code, COUNT(*) FROM "EmployeeInfoTab"
GROUP BY code HAVING COUNT(*) > 1;
```

**解决**:
```sql
-- 删除大写代码的重复页签
DELETE FROM "EmployeeInfoTabField"
WHERE "tabId" IN (SELECT id FROM "EmployeeInfoTab" WHERE code IN ('BASIC_INFO', 'WORK_INFO'));

DELETE FROM "EmployeeInfoTabGroup"
WHERE "tabId" IN (SELECT id FROM "EmployeeInfoTab" WHERE code IN ('BASIC_INFO', 'WORK_INFO'));

DELETE FROM "EmployeeInfoTab"
WHERE code IN ('BASIC_INFO', 'WORK_INFO');
```

### 问题3: 孤立的子表记录

**症状**: 数据完整性错误

**诊断**:
```sql
SELECT 'EmployeeEducation' AS table, COUNT(*) AS count
FROM "EmployeeEducation" e
WHERE NOT EXISTS (SELECT 1 FROM "Employee" emp WHERE emp.id = e."employeeId")
UNION ALL
SELECT 'EmployeeWorkExperience', COUNT(*)
FROM "EmployeeWorkExperience" e
WHERE NOT EXISTS (SELECT 1 FROM "Employee" emp WHERE emp.id = e."employeeId");
```

**解决**:
```sql
-- 删除孤立记录
DELETE FROM "EmployeeEducation"
WHERE NOT EXISTS (SELECT 1 FROM "Employee" WHERE "Employee".id = "EmployeeEducation"."employeeId");

DELETE FROM "EmployeeWorkExperience"
WHERE NOT EXISTS (SELECT 1 FROM "Employee" WHERE "Employee".id = "EmployeeWorkExperience"."employeeId");
```
