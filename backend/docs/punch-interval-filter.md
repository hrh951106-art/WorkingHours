# 摆卡间隔过滤算法说明

## 功能概述

摆卡间隔过滤算法用于处理重复刷卡数据，在成对收卡之前对刷卡记录进行筛选，过滤掉在短时间内的重复刷卡。

## 业务场景

在实际使用中，可能出现以下情况：
1. 员工在考勤机上连续刷多次卡
2. 打卡数据没有进出标记（punchType 为空）
3. 打卡设备没有绑定劳动力账户（accountId 为空）

本算法在成对收卡之前，对刷卡记录进行预处理，过滤掉重复数据。

## 过滤规则

### 前置条件

1. **劳动力账户不同** → 不比较，全部保留
2. **强制补卡**（source = 'MANUAL_SUPPLEMENT'）→ 不受间隔影响，全部保留

### 比较规则（劳动力账户相同时）

| 进出标记类型 | 是否比较 | 比较规则 |
|-------------|---------|---------|
| 进 + 进 | ✅ 是 | 动态参考点（保留的进卡为新参考点） |
| 出 + 出 | ✅ 是 | 动态参考点（保留的出卡为新参考点） |
| 进 + 出 | ❌ 否 | 全部保留 |
| 有标记 + 无标记 | ❌ 否 | 全部保留 |
| 无标记 + 无标记 | ✅ 是 | 动态参考点 |

### 动态参考点算法

1. 第一张卡直接保留，作为参考点
2. 后续卡与参考点比较：
   - 如果需要比较且间隔 < 摆卡间隔 → 丢弃当前卡（参考点不变）
   - 如果需要比较且间隔 ≥ 摆卡间隔 → 保留当前卡，更新参考点
   - 如果不需要比较 → 保留当前卡，更新参考点

## 配置说明

摆卡间隔在**打卡规则配置**中设置：

1. 进入 `计算管理` → `打卡规则`
2. 新增或编辑规则
3. 在 `设备组与摆卡间隔配置` 中配置：
   - **设备组**：选择要应用的设备组
   - **摆卡间隔（分钟）**：设置过滤间隔
     - `0` 表示不启用过滤
     - `>0` 表示启用过滤，例如设置为 `5`，则 5 分钟内的重复刷卡会被过滤

## 示例场景

### 场景 1：连续刷进卡

**配置**：摆卡间隔 = 5 分钟

**刷卡记录**：
```
08:00:00 - 进 (账户A)
08:01:30 - 进 (账户A)
08:03:00 - 进 (账户A)
08:10:00 - 出 (账户A)
```

**过滤结果**：
```
08:00:00 - 进 (账户A) ✓ 保留，作为参考点
08:01:30 - 进 (账户A) ✗ 丢弃（与参考点间隔 1.5 分钟 < 5 分钟）
08:03:00 - 进 (账户A) ✗ 丢弃（与参考点间隔 3 分钟 < 5 分钟）
08:10:00 - 出 (账户A) ✓ 保留（进+出不比较）
```

### 场景 2：进出混合

**配置**：摆卡间隔 = 5 分钟

**刷卡记录**：
```
08:00:00 - 进 (账户A)
08:00:30 - 进 (账户A)
08:05:00 - 出 (账户A)
08:05:30 - 出 (账户A)
```

**过滤结果**：
```
08:00:00 - 进 (账户A) ✓ 保留
08:00:30 - 进 (账户A) ✗ 丢弃（与参考点间隔 0.5 分钟 < 5 分钟）
08:05:00 - 出 (账户A) ✓ 保留（进+出不比较，成为新参考点）
08:05:30 - 出 (账户A) ✗ 丢弃（与参考点间隔 0.5 分钟 < 5 分钟）
```

### 场景 3：不同账户

**配置**：摆卡间隔 = 5 分钟

**刷卡记录**：
```
08:00:00 - 进 (账户A)
08:00:30 - 进 (账户B)
08:01:00 - 进 (账户A)
```

**过滤结果**：
```
08:00:00 - 进 (账户A) ✓ 保留（账户不同，不过滤）
08:00:30 - 进 (账户B) ✓ 保留（账户不同，不过滤）
08:01:00 - 进 (账户A) ✓ 保留（账户A参考点为08:00:00，间隔1分钟 < 5分钟，但账户B卡不改变参考点）
```

### 场景 4：强制补卡

**配置**：摆卡间隔 = 5 分钟

**刷卡记录**：
```
08:00:00 - 进 (账户A, source=AUTO)
08:00:30 - 进 (账户A, source=MANUAL_SUPPLEMENT)
08:01:00 - 进 (账户A, source=AUTO)
```

**过滤结果**：
```
08:00:00 - 进 (账户A) ✓ 保留
08:00:30 - 进 (账户A) ✓ 保留（强制补卡不受间隔影响）
08:01:00 - 进 (账户A) ✗ 丢弃（参考点变为08:00:30，间隔0.5分钟 < 5分钟）
```

## 技术实现

### 服务类

- **文件**：`src/modules/punch/pairing.service.ts`
- **方法**：`createPunchPairsForAccount`
- **调用位置**：在成对收卡之前自动调用

### 日志输出

摆卡过滤会输出日志，格式如下：
```
[摆卡间隔过滤] 员工: E001, 日期: 2025-01-01, 账户: 100, 过滤前: 15张卡, 过滤后: 8张卡, 间隔: 5分钟
```

## 注意事项

1. **punchType 为空的情况**：
   - 如果所有卡的 punchType 都为空，会被视为"无标记+无标记"，需要进行比较
   - 如果部分卡有标记，部分卡无标记，则被视为"有标记+无标记"，不进行比较

2. **accountId 为空的情况**：
   - accountId 为 null 的记录会被归为一组（"无账户组"）
   - 无账户组与有账户组之间不进行比较

3. **过滤顺序**：
   - 摆卡间隔过滤在账户合并之后、成对收卡之前执行
   - 过滤后的记录才会进入成对收卡逻辑

4. **配对间隔与过滤间隔**：
   - 当前实现复用 `pairingInterval` 配置
   - 如果未来需要分别配置，可以新增 `filterInterval` 字段
