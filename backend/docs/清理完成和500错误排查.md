## 🛠️ 可能的解决方案

### 方案1：排除新字段（推荐）

前端提交时，不提交 `isSystem` 和 `dataSourceId` 字段：

```typescript
// 前端代码
const submitData = {
  ...formData,
  // 排除这两个字段
  isSystem: undefined,
  dataSourceId: undefined,
};

await api.post('/api/hr/employees', submitData);
```

### 方案2：后端添加字段过滤

在后端创建/更新方法中，过滤掉只读字段：

```typescript
async create(dto: CreateEmployeeDto) {
  // 过滤掉前端可能提交的只读字段
  const { isSystem, dataSourceId, ...cleanDto } = dto as any;

  return this.prisma.employee.create({
    data: cleanDto
  });
}
```

### 方案3：添加字段到DTO

如果前端确实需要提交这些字段，需要在DTO中添加：

```typescript
export class CreateEmployeeDto {
  // ... 其他字段

  @IsOptional()
  @IsBoolean()
  isSystem?: boolean;

  @IsOptional()
  @IsInt()
  dataSourceId?: number;
}
```
