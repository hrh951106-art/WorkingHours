# 修复方案对比说明

> **日期**: 2026-04-01
> **问题**: 生产环境字段不显示，无法使用数据源

---

## 📊 问题回顾

### 用户反馈
1. 使用方案A后，字段显示了
2. 但无法选择数据源，没有下拉框
3. 需要完整的解决方案

### 原因分析

```
方案A做了什么：
├─ 将 fieldType = SELECT_SINGLE 改为 SYSTEM
├─ 字段能显示了（因为前端识别SYSTEM）
└─ 但SYSTEM类型不关联CustomField → 无法使用数据源

完整方案需要：
├─ fieldType = SELECT_SINGLE
├─ 创建CustomField记录
├─ 关联DataSource
├─ 部署支持SELECT_SINGLE的前端
└─ 结果：字段显示 + 下拉框 + 数据源选项
```

---

## 🆚 两种方案对比

### 方案A：快速修复（临时方案）

```sql
UPDATE "EmployeeInfoTabField"
SET "fieldType" = 'SYSTEM'
WHERE "fieldType" = 'SELECT_SINGLE'
AND "fieldCode" IN ('gender', 'nation', ...);
```

**执行时间**: 5分钟

**优点**:
- ✅ 快速恢复字段显示
- ✅ 不需要更新前端代码
- ✅ 风险小

**缺点**:
- ❌ **字段变成输入框，不是下拉框**
- ❌ **无法使用数据源功能**
- ❌ **需要手动输入，容易出错**
- ❌ **只是临时方案，后续还需完整修复**

**适用场景**:
- 生产故障需要紧急恢复
- 作为完整修复前的临时措施
- 不需要数据源功能的场景

**用户体验**: ⭐⭐ (字段显示但无下拉框)

---

### 完整方案：彻底修复（推荐方案）

```sql
-- Step 1: 创建CustomField记录
INSERT INTO "CustomField" (code, name, type, dataSourceId, ...)
SELECT fieldCode, fieldName, 'SELECT_SINGLE', ds.id, ...
FROM "EmployeeInfoTabField" eif
JOIN "DataSource" ds ON ds.code = eif.fieldCode
WHERE ...;

-- Step 2: 统一类型为SELECT_SINGLE
UPDATE "EmployeeInfoTabField"
SET "fieldType" = 'SELECT_SINGLE'
WHERE fieldCode IN (...);

-- Step 3: 部署支持SELECT_SINGLE的前端
```

**执行时间**: 30分钟

**优点**:
- ✅ **字段显示为下拉框**
- ✅ **可以使用数据源**
- ✅ **用户可以选择，不会出错**
- ✅ **支持数据源中的任何值（包括"03"等代码）**
- ✅ **长期稳定的解决方案**

**缺点**:
- ⚠️ 需要同时更新前端代码
- ⚠️ 执行步骤较多
- ⚠️ 需要重启前端服务

**适用场景**:
- ✅ **正式生产环境修复**
- ✅ **需要完整功能的场景**
- ✅ **有足够维护时间的场景**

**用户体验**: ⭐⭐⭐⭐⭐ (完整的下拉框功能)
