## ⚠️ 当前问题

### Prisma Client 编译问题

**问题**：Prisma schema provider 是 "postgresql"，但开发环境使用 SQLite

**错误信息**：
```
Validation Error Count: 1
at t (/Users/aaron.he/Desktop/AI/JY/backend/node_modules/@prisma/client/runtime/library.js:112:2488)
```

**原因**：
- Prisma schema 配置为 PostgreSQL
- .env 中 DATABASE_URL 是 SQLite (file:./dev.db)
- 新添加的字段通过脚本直接添加到 SQLite 数据库
- Prisma Client 与实际数据库结构不匹配

**影响**：后端服务无法启动

## 🚀 解决方案

### 方案1：修改 Prisma Schema 支持 SQLite（开发环境）

#### 1.1 修改 schema.prisma

```prisma
datasource db {
  provider = "sqlite"  // 改为 sqlite
  url      = env("DATABASE_URL")
}
```

#### 1.2 调整字段类型（SQLite vs PostgreSQL 差异）

SQLite 使用 `Boolean` 作为类型，但底层是 INTEGER：
```prisma
isSystem     Boolean               @default(false)  // SQLite会存储为 0/1
```

#### 1.3 重新生成 Prisma Client

```bash
npx prisma generate
```

#### 1.4 重启服务

```bash
npm run start:dev
```

### 方案2：创建 PostgreSQL 开发环境（推荐）

#### 2.1 安装 PostgreSQL

```bash
# macOS
brew install postgresql
brew services start postgresql

# 创建数据库
createdb jy_development
```

#### 2.2 修改 .env

```env
DATABASE_URL="postgresql://jy_admin@localhost:5432/jy_development"
```

#### 2.3 运行迁移

```bash
npx prisma migrate dev
```

#### 2.4 生成 Prisma Client

```bash
npx prisma generate
```

#### 2.5 重启服务

```bash
npm run start:dev
```

### 方案3：使用现有 PostgreSQL 镜像（最快）

#### 3.1 启动 PostgreSQL Docker 容器

```bash
docker run --name jy-postgres \
  -e POSTGRES_USER=jy_admin \
  -e POSTGRES_PASSWORD=jy_admin \
  -e POSTGRES_DB=jy_development \
  -p 5432:5432 \
  -d postgres:16
```

#### 3.2 修改 .env

```env
DATABASE_URL="postgresql://jy_admin:jy_admin@localhost:5432/jy_development"
```

#### 3.3 执行生产环境脚本

```bash
psql -U jy_admin -d jy_development \
  -f scripts/add-datasource-to-system-fields-prod.sql
```

#### 3.4 生成 Prisma Client 并重启

```bash
npx prisma generate
npm run start:dev
```

## 🎯 核心改动回顾

### 数据结构变更

**之前（错误方案）**：
```
系统内置字段
  ↓ (错误创建)
CustomField
  ↓ dataSourceId
DataSource
```

**现在（正确方案）**：
```
系统内置字段
EmployeeInfoTabField (isSystem=true)
  ↓ dataSourceId (直接关联)
DataSource

自定义字段
EmployeeInfoTabField (isSystem=false)
  ↓ fieldCode
CustomField
  ↓ dataSourceId
DataSource
```

### 后端逻辑变更

**之前**：
```typescript
// 只处理 CUSTOM 类型
if (field.fieldType === 'CUSTOM') {
  const customField = customFields.find(...);
  return { ...field, dataSource: customField?.dataSource };
}
```

**现在**：
```typescript
// 处理系统内置字段
if (field.isSystem) {
  return { ...field, dataSource: field.dataSource };
}

// 处理自定义字段
if (field.fieldType === 'CUSTOM') {
  const customField = customFields.find(...);
  return { ...field, dataSource: customField?.dataSource };
}
```
