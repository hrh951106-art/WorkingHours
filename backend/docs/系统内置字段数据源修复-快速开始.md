### 第1步：修改后端代码

修改文件：`src/modules/hr/employee-info-tab.service.ts`

找到 `enrichFieldWithType` 方法（约125行），修改为：

```typescript
private enrichFieldWithType(field: any, customFields: any[]) {
  // CUSTOM字段：从CustomField中查找数据源
  if (field.fieldType === 'CUSTOM') {
    const customField = customFields.find((cf) => cf.code === field.fieldCode);
    return {
      ...field,
      type: customField?.type || 'TEXT',
      dataSource: customField?.dataSource || null,
    };
  }

  // SELECT字段：也尝试从CustomField中查找数据源（新增逻辑）
  if (field.fieldType === 'SELECT') {
    const customField = customFields.find((cf) => cf.code === field.fieldCode);

    if (customField?.dataSource) {
      return {
        ...field,
        type: 'SELECT',
        dataSource: customField.dataSource,
      };
    }
  }

  return field;
}
```

**关键改动**：添加了对 `fieldType === 'SELECT'` 的处理

## ❓ 常见问题

### Q1: 为什么需要修改后端代码？

**A**: 原有代码只处理 `fieldType='CUSTOM'` 的字段，而系统内置字段是 `fieldType='SELECT'`。修改后，SELECT 类型的字段也能从 CustomField 获取数据源。

### Q2: 如果不想修改后端代码怎么办？

**A**: 可以使用之前创建的 SQL 视图方案（`scripts/quick-fix-datasource-view.sql`），不需要修改代码即可生效。

### Q3: 前端显示还是空白？

**A**: 检查以下几点：
1. 后端代码是否已修改
2. 后端服务是否已重启
3. DataSourceOption 表是否有数据
4. 浏览器控制台是否有错误

### Q4: 如何回滚？

**A**: 使用 Git 恢复代码，或使用数据库备份恢复数据：
```bash
# 代码回滚
git checkout src/modules/hr/employee-info-tab.service.ts

# 数据恢复
psql -U jy_admin -d jy_production < backup_20250331.sql
```
