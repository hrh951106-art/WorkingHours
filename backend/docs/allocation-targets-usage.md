# 分摊目标（AllocationRuleTarget）的作用说明

## 概述

分摊目标（AllocationRuleTarget）用于定义间接工时分摊到哪些具体的产线或账户。

## 两种分摊模式

代码中存在两种分摊逻辑模式：

### 1. 手动分摊模式（旧版逻辑）

**位置**：`executeAllocationCalculation` 方法（约第2657行）

**使用方式**：
- **必须配置分摊目标**：如果没有配置目标（`rule.targets.length === 0`），会跳过该规则
- **分摊目标决定分摊到哪里**：遍历 `rule.targets`，将工时分摊到每个目标
- **权重决定分摊比例**：根据 `target.weight` 来计算分摊比例

**规则类型**：
- `PROPORTIONAL`（按比例）：分摊比例 = 该目标权重 / 总权重
- `EQUAL`（平均）：分摊比例 = 1 / 目标数量
- `FIXED`（固定）：分摊比例 = 该目标权重 / 待分摊工时

**示例场景**：
```
配置：将李四的8.5h间接工时分摊到产线
规则：
  - 目标1: L1产线，权重 60%
  - 目标2: L2产线，权重 40%

结果：
  - L1产线: 8.5 × 0.6 = 5.1h
  - L2产线: 8.5 × 0.4 = 3.4h
```

**适用情况**：
- 需要精确控制每个目标的分摊比例
- 目标数量固定，不随生产情况变化
- 手动管理模式

### 2. 自动分摊模式（新版逻辑）

**位置**：
- `executeActualHoursAllocation` - 按实际工时自动分摊
- `executeActualYieldAllocation` - 按实际产量自动分摊
- `executeEquivalentYieldAllocation` - 按同效产量自动分摊
- `executeStandardHoursAllocation` - 按标准工时自动分摊

**使用方式**：
- **不使用配置的分摊目标**：这些方法中完全没有使用 `rule.targets`
- **自动获取分摊目标**：通过 `shiftLines` 自动获取当天开线的产线
- **根据实际数据计算分摊比例**：根据产量/工时数据动态计算

**示例场景**：
```
配置：将李四的8.5h间接工时按实际产量自动分摊
规则：不配置目标（或配置目标但不使用）

当天开线情况：
  - L1产线开线，生产3000件
  - L2产线开线，生产2000件

自动计算：
  - L1产线分摊比例 = 3000 / (3000+2000) = 60%
  - L2产线分摊比例 = 2000 / (3000+2000) = 40%

结果：
  - L1产线: 8.5 × 0.6 = 5.1h
  - L2产线: 8.5 × 0.4 = 3.4h
```

**适用情况**：
- 分摊比例需要根据实际生产数据动态调整
- 目标（产线）可能随时变化
- 自动化管理模式

## 当前实现的问题

### 问题1：分摊目标的配置未被使用

在我们实现的四种分摊方式中：
1. ✅ ACTUAL_HOURS（按实际工时）- 使用自动模式，遍历 shiftLines
2. ✅ ACTUAL_YIELDS（按实际产量）- 使用自动模式，遍历 shiftLines
3. ✅ EQUIVALENT_YIELDS（按同效产量）- 使用自动模式，遍历 shiftLines
4. ✅ STANDARD_HOURS（按标准工时）- 使用自动模式，遍历 shiftLines

**所有四种方式都没有使用 `rule.targets`**，这意味着：
- 即使在界面上配置了分摊目标，也不会被使用
- 分摊完全由当天开线的产线决定

### 问题2：配置了目标但未检查

当前代码在执行分摊时，没有检查：
- `rule.targets.length === 0` - 没有这个检查
- 目标数量与实际开线产线数量是否一致
- 目标的有效性

### 问题3：两种模式并存导致混淆

- 界面上要求配置分摊目标
- 但实际执行时完全不用这些目标
- 用户可能误以为配置的目标会影响分摊结果

## 建议的改进方案

### 方案1：完全移除分摊目标的依赖

如果决定采用自动分摊模式，应该：
1. 在前端界面中移除"分摊目标"的配置要求
2. 或者将"分摊目标"改为可选配置
3. 在说明文档中明确：分摊目标由系统自动根据开线情况确定

### 方案2：支持两种模式

让用户选择使用哪种模式：
1. **手动模式**：使用配置的分摊目标和权重
2. **自动模式**：根据实际数据自动计算分摊比例

配置时添加模式选择：
```typescript
enum AllocationMode {
  MANUAL,    // 手动模式：使用 rule.targets
  AUTO       // 自动模式：根据实际数据自动分摊
}
```

### 方案3：混合模式

结合两种模式的优点：
1. 配置分摊目标作为"候选目标池"
2. 系统自动从候选池中筛选出当天实际开线的产线
3. 根据实际数据计算分摊比例

这样既保证了灵活性，又支持自动化。

|---------|-------------|------------|------------|

| ACTUAL_YIELDS | ❌ 否 | shiftLines（自动） | 实际产量比例 |
| EQUIVALENT_YIELDS | ❌ 否 | shiftLines（自动） | 标准产量比例 |
