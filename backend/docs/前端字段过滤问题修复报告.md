# 前端字段过滤问题修复报告

> **修复时间**: 2025-04-01 11:14
> **问题**: 前端过滤掉SELECT_SINGLE类型字段，导致下拉字段不显示

---

## 🔍 问题定位

### 问题代码位置
**文件**: `frontend/src/pages/hr/EmployeeCreatePage.tsx`
**行数**: 1287-1292

### 错误的过滤逻辑

```typescript
// ❌ 错误：只包含SYSTEM类型
const systemFields = group.fields?.filter((f: any) =>
  f.fieldType === 'SYSTEM' && !f.isHidden
) || [];
```

**问题分析**:
- gender、nation等字段的fieldType是`SELECT_SINGLE`
- 前端只过滤`fieldType === 'SYSTEM'`的字段
- 所有SELECT_SINGLE类型的字段被过滤掉
- 结果：这些字段不会渲染到页面上

---

## ✅ 修复方案

### 修复后的代码

```typescript
// ✅ 正确：包含SYSTEM、SELECT_SINGLE、SELECT_MULTI
const systemFields = group.fields?.filter((f: any) =>
  (f.fieldType === 'SYSTEM' || f.fieldType === 'SELECT_SINGLE' || f.fieldType === 'SELECT_MULTI') && !f.isHidden
) || [];
```

### 修复说明

1. **添加SELECT_SINGLE类型判断**
   - 原来只包含SYSTEM类型
   - 现在同时包含SYSTEM、SELECT_SINGLE、SELECT_MULTI类型

2. **保持其他逻辑不变**
   - CUSTOM类型字段的处理不变
   - 隐藏字段过滤逻辑不变
