# 系统内置字段数据源关联 - 正确方案

> **修改时间**: 2025-03-31
> **问题**: 系统内置字段不应该通过 CustomField 处理
> **方案**: 系统内置字段直接在 EmployeeInfoTabField 关联数据源

---

## 📋 问题分析

### 之前方案的问题

1. ❌ **混淆概念**: 将系统内置字段创建为 CustomField 记录
2. ❌ **删除风险**: CustomField 记录可以在配置页面被删除
3. ❌ **逻辑混乱**: CustomField 应该只存储用户自定义的字段
4. ❌ **标识不清**: 无法区分系统内置字段和自定义字段

### 正确的设计理念

**系统内置字段**：
- 预定义的字段（如：性别、民族、学历等）
- 不允许删除
- 直接在 EmployeeInfoTabField 中关联数据源
- 通过 `isSystem = true` 标识

**自定义字段**：
- 用户自己创建的字段
- 允许删除和修改
- 通过 CustomField 表管理
- 通过 `isSystem = false` 标识

```prisma
  group        EmployeeInfoTabGroup? @relation(fields: [groupId], references: [id])

  @@index([groupId])
```

添加反向关系：


#### 2. 修改字段数据源处理逻辑

```typescript
private enrichFieldWithType(field: any, customFields: any[]) {
  // ✨ 系统内置字段：直接使用 EmployeeInfoTabField.dataSource
  if (field.isSystem) {
    return {
      ...field,
      type: field.fieldType === 'SELECT' ? 'SELECT' : 'TEXT',
      dataSource: field.dataSource || null,
    };
  }

  // 自定义字段：从 CustomField 中查找数据源
  if (field.fieldType === 'CUSTOM') {
    const customField = customFields.find((cf) => cf.code === field.fieldCode);
    return {
      ...field,
      type: customField?.type || 'TEXT',
      dataSource: customField?.dataSource || null,
    };
  }

  // 其他类型字段：直接返回
  return field;
}
```

## ⚠️ 注意事项

### 1. CustomField 表的用途

CustomField 表只用于：
- 用户自己创建的自定义字段
- 需要额外配置的字段（如：默认值、分组等）

CustomField 表不用于：
- 系统内置字段（如：性别、民族等）

### 2. fieldType 的含义

- `SYSTEM`: 系统内置字段（旧设计，已废弃）
- `CUSTOM`: 自定义字段
- `SELECT`: 下拉选择字段
- `TEXT`: 文本字段
- `DATE`: 日期字段
- 等等...

**重要**: `isSystem` 字段才是标识系统内置字段的正确方式

### 3. 向后兼容

如果之前有系统内置字段的 CustomField 记录：
- 脚本已自动删除
- 数据已迁移到 EmployeeInfoTabField.dataSourceId
- 不影响现有功能
