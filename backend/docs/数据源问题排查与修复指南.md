# 系统内置字段数据源问题排查与修复指南

> **问题描述**: 生产环境中，系统内置的下拉字段（如性别、学历等）无法从查找项中获取数据源数据
> **更新日期**: 2025-03-31

---

## 🔍 问题根本原因分析

### 代码逻辑分析

查看 `src/modules/hr/employee-info-tab.service.ts` 中的 `enrichFieldWithType` 方法：

```typescript
private enrichFieldWithType(field: any, customFields: any[]) {
  if (field.fieldType === 'CUSTOM') {
    const customField = customFields.find((cf) => cf.code === field.fieldCode);
    return {
      ...field,
      type: customField?.type || 'TEXT',
      dataSource: customField?.dataSource || null,  // ← 只在CUSTOM时查询数据源
    };
  }
  return field;  // ← SYSTEM字段直接返回，没有dataSource属性
}
```

**问题所在**:
1. ✅ 代码只在 `fieldType === 'CUSTOM'` 时才查询数据源
2. ❌ 系统内置字段的 `fieldType` 是 `'SYSTEM'`，不是 `'CUSTOM'`
3. ❌ 系统内置字段返回时没有 `dataSource` 属性
4. ❌ 前端获取不到数据源选项

### 数据流程图

```
正常流程（CUSTOM字段）:
EmployeeInfoTabField (fieldType = 'CUSTOM')
    ↓ 查询
CustomField (包含dataSourceId)
    ↓ 关联
DataSource + DataSourceOption
    ↓ 返回给前端
前端显示下拉选项

问题流程（SYSTEM字段）:
EmployeeInfoTabField (fieldType = 'SYSTEM')
    ↓ 不查询（代码逻辑跳过）
没有数据源信息 ❌
    ↓ 返回给前端
前端无法显示下拉选项 ❌
```

## 📋 解决方案

### 方案1：修改EmployeeInfoTabField配置（推荐）

**步骤1**: 将需要数据源的SYSTEM字段改为SELECT类型

```sql

  AND "tabId" = (SELECT id FROM "EmployeeInfoTab" WHERE code = 'basic_info');

-- 更新nation字段类型
UPDATE "EmployeeInfoTabField"
SET fieldType = 'SELECT'
WHERE fieldCode = 'nation'
  AND "tabId" = (SELECT id FROM "EmployeeInfoTab" WHERE code = 'basic_info');

-- 更新maritalStatus字段类型
UPDATE "EmployeeInfoTabField"
SET fieldType = 'SELECT'
WHERE fieldCode = 'maritalStatus'
  AND "tabId" = (SELECT id FROM "EmployeeInfoTab" WHERE code = 'basic_info');

-- 更新politicalStatus字段类型
UPDATE "EmployeeInfoTabField"
SET fieldType = 'SELECT'
WHERE fieldCode = 'politicalStatus'
  AND "tabId" = (SELECT id FROM "EmployeeInfoTab" WHERE code = 'basic_info');

-- 更新educationLevel字段类型
UPDATE "EmployeeInfoTabField"
SET fieldType = 'SELECT'
WHERE fieldCode = 'educationLevel'
  AND "tabId" = (SELECT id FROM "EmployeeInfoTab" WHERE code = 'education_info');
```

### 方案2：修改后端代码逻辑（不推荐）

如果不想修改数据库配置，可以修改后端代码逻辑，让SYSTEM类型的字段也能获取数据源。

**修改文件**: `src/modules/hr/employee-info-tab.service.ts`

```typescript
private enrichFieldWithType(field: any, customFields: any[]) {
  // CUSTOM字段：从CustomField中查找数据源
  if (field.fieldType === 'CUSTOM') {
    const customField = customFields.find((cf) => cf.code === field.fieldCode);
    return {
      ...field,
      type: customField?.type || 'TEXT',
      dataSource: customField?.dataSource || null,
    };
  }

  // SYSTEM字段：检查是否需要数据源（新增逻辑）
  if (field.fieldType === 'SYSTEM') {
    const systemFieldsNeedingDataSource = [
      'gender', 'nation', 'maritalStatus', 'politicalStatus',
      'educationLevel', 'employeeType'
    ];

    if (systemFieldsNeedingDataSource.includes(field.fieldCode)) {
      const customField = customFields.find((cf) => cf.code === field.fieldCode);
      return {
        ...field,
        type: 'SELECT',
        dataSource: customField?.dataSource || null,
      };
    }
  }

  return field;
}
```

**注意**: 修改代码后需要重新构建和部署应用。
