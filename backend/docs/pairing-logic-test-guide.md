# 摆卡逻辑测试指南

## 概述

本文档用于验证摆卡逻辑的正确性，包括摆卡间隔过滤和成对收卡功能。

## 测试环境准备

### 1. 确保数据库中有测试数据

```sql
-- 检查员工数据
SELECT * FROM Employee WHERE employeeNo = 'TEST001';

-- 检查打卡记录
SELECT * FROM PunchRecord WHERE employeeNo = 'TEST001' ORDER BY punchTime;

-- 检查摆卡记录
SELECT * FROM PunchPair WHERE employeeNo = 'TEST001' ORDER BY pairDate;
```

### 2. 测试API端点

```bash
# 单个员工摆卡
curl -X POST http://localhost:3000/api/punch/pairing \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "employeeNo": "TEST001",
    "pairDate": "2025-01-15"

# 批量摆卡

  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "pairDate": "2025-01-15"
  }'
```

## 测试场景

### 场景1：基本摆卡 - 无过滤

**前置条件**：
- 摆卡间隔设置为 0（不启用过滤）
- 员工有4条打卡记录：2进2出

**输入数据**：
```
08:00:00 - 进 (账户A)
08:05:00 - 进 (账户A)
12:00:00 - 出 (账户A)
12:05:00 - 出 (账户A)
```

**预期结果**：
- 生成2对摆卡记录：
  - 对1：08:00:00 (进) + 12:00:00 (出)
  - 对2：08:05:00 (进) + 12:05:00 (出)

**验证SQL**：
```sql
SELECT
  id,
  inPunchTime,
  outPunchTime,
  workHours,
  accountId
FROM PunchPair
WHERE employeeNo = 'TEST001'
  AND pairDate = '2025-01-15'
ORDER BY inPunchTime;
```

### 场景2：摆卡间隔过滤 - 连续进卡

**前置条件**：
- 摆卡间隔设置为 5 分钟
- 员工有多条连续进卡

**输入数据**：
```
08:00:00 - 进 (账户A)
08:01:30 - 进 (账户A)
08:03:00 - 进 (账户A)
08:10:00 - 出 (账户A)
```

**预期结果**：
- 过滤后保留：
  - 08:00:00 - 进 (账户A) ✓ 保留
  - 08:01:30 - 进 (账户A) ✗ 丢弃（间隔1.5分钟 < 5分钟）
  - 08:03:00 - 进 (账户A) ✗ 丢弃（间隔3分钟 < 5分钟）
  - 08:10:00 - 出 (账户A) ✓ 保留
- 生成1对摆卡记录：
  - 对1：08:00:00 (进) + 08:10:00 (出)

**日志验证**：
```
[摆卡间隔过滤] 员工: TEST001, 日期: 2025-01-15, 账户: 100, 过滤前: 4张卡, 过滤后: 2张卡, 间隔: 5分钟
```

### 场景3：不同账户不过滤

**前置条件**：
- 摆卡间隔设置为 5 分钟
- 员工有不同账户的打卡记录

**输入数据**：
```
08:00:00 - 进 (账户A)
08:00:30 - 进 (账户B)
08:01:00 - 进 (账户A)
```

**预期结果**：
- 所有记录都保留（账户不同，不过滤）
- 生成2对摆卡记录：
  - 账户A：08:00:00 (进) 单卡
  - 账户B：08:00:30 (进) 单卡

### 场景4：强制补卡不受间隔影响

**前置条件**：
- 摆卡间隔设置为 5 分钟
- 包含手动补卡记录

**输入数据**：
```
08:00:00 - 进 (账户A, source=AUTO)
08:00:30 - 进 (账户A, source=MANUAL_SUPPLEMENT)
08:01:00 - 进 (账户A, source=AUTO)
```

**预期结果**：
- 过滤后保留：
  - 08:00:00 - 进 (账户A, AUTO) ✓ 保留
  - 08:00:30 - 进 (账户A, MANUAL_SUPPLEMENT) ✓ 保留（强制补卡）
  - 08:01:00 - 进 (账户A, AUTO) ✗ 丢弃（与参考点08:00:30间隔0.5分钟 < 5分钟）
- 生成1对摆卡记录：
  - 对1：08:00:00 (进) + 08:00:30 (出) 或单卡

### 场景5：进出混合过滤

**前置条件**：
- 摆卡间隔设置为 5 分钟

**输入数据**：
```
08:00:00 - 进 (账户A)
08:00:30 - 进 (账户A)
08:05:00 - 出 (账户A)
08:05:30 - 出 (账户A)
```

**预期结果**：
- 过滤后保留：
  - 08:00:00 - 进 (账户A) ✓ 保留
  - 08:00:30 - 进 (账户A) ✗ 丢弃（与参考点08:00:00间隔0.5分钟 < 5分钟）
  - 08:05:00 - 出 (账户A) ✓ 保留（进+出不比较，成为新参考点）
  - 08:05:30 - 出 (账户A) ✗ 丢弃（与参考点08:05:00间隔0.5分钟 < 5分钟）
- 生成1对摆卡记录：
  - 对1：08:00:00 (进) + 08:05:00 (出)

### 场景6：设备组级别配置

**前置条件**：
- 设备组A：摆卡间隔 5 分钟
- 设备组B：摆卡间隔 10 分钟
- 员工在不同设备组有打卡记录

**输入数据**：
```
08:00:00 - 进 (账户A, 设备组A)
08:03:00 - 进 (账户A, 设备组A)  # 间隔3分钟 < 5分钟，应丢弃
08:10:00 - 进 (账户A, 设备组B)  # 距上次保留10分钟 >= 5分钟，应保留
08:12:00 - 进 (账户A, 设备组B)  # 间隔2分钟 < 10分钟，应丢弃
```

**预期结果**：
- 过滤后保留：
  - 08:00:00 - 进 (账户A, 设备组A) ✓ 保留
  - 08:03:00 - 进 (账户A, 设备组A) ✗ 丢弃（间隔3分钟 < 5分钟）
  - 08:10:00 - 进 (账户A, 设备组B) ✓ 保留（间隔10分钟 >= 5分钟）
  - 08:12:00 - 进 (账户A, 设备组B) ✗ 丢弃（间隔2分钟 < 10分钟）

## 已知问题和注意事项

1. **punchType 为空的处理**
   - 如果所有卡的 punchType 都为空，会被视为"无标记+无标记"，需要进行比较
   - 如果部分卡有标记，部分卡无标记，则被视为"有标记+无标记"，不进行比较

2. **accountId 为空的处理**
   - accountId 为 null 的记录会被归为一组（"无账户组"）
   - 无账户组与有账户组之间不进行比较

3. **摆卡间隔与配对间隔**
   - 当前实现中，过滤间隔和配对间隔使用相同值
   - 过滤：在成对收卡之前，过滤掉重复打卡
   - 配对：在成对收卡时，确定进出卡的最小时间间隔

4. **配置优先级**
   - 新的关联表 `PunchRuleDeviceGroupInterval` 优先级高于旧的 `configs` JSON字段
   - 系统会先尝试从关联表读取配置，如果没有则从 configs 字段读取
