# 系统内置字段数据源快速修复 - 快速开始

> **5分钟修复生产环境中系统内置字段无法获取数据源的问题**

---

## 🎯 问题回顾

**现象**: 员工信息编辑页面中，性别、民族、婚姻状况等下拉框为空，无法选择选项。

**原因**: 后端代码只处理 `fieldType='CUSTOM'` 的字段，而系统内置字段是 `fieldType='SYSTEM'`，导致无法获取数据源。

**解决方案**: 创建SQL视图和新API端点，绕过代码限制，直接从数据库获取完整数据源。

## ⚠️ 注意事项

### 1. 数据库依赖

此方案依赖PostgreSQL的以下特性：
- VIEW（视图）
- FUNCTION（函数）
- JSON/JSONB类型
- jsonb_build_object、json_agg等函数

确保生产环境使用PostgreSQL 9.5+

### 2. 性能考虑

- SQL视图每次都执行完整查询，适合中小规模数据
- 如果页签字段数超过1000，建议添加缓存
- 如果响应时间>500ms，参考部署指南中的性能优化部分

### 3. 兼容性

- ✅ 不影响现有的 `/api/hr/employee-info-tabs/display` API
- ✅ 新API可以和旧API共存
- ✅ 前端可以选择使用新旧API

### 4. 数据要求

此方案需要以下数据存在：
- ✅ `EmployeeInfoTab` 表有页签数据
- ✅ `EmployeeInfoTabField` 表有字段数据
- ✅ `CustomField` 表有字段配置（需要数据源的字段）
- ✅ `DataSource` 表有数据源定义
- ✅ `DataSourceOption` 表有选项数据

如果缺失，先运行 `scripts/fix-datasource-fields.sql`

## 🐛 故障排查

### 问题1: API返回404

**检查**:
```bash
# 检查路由是否注册
curl -X GET http://localhost:3001/api/hr/employee-info-tabs-quickfix/validate/datasource
```

**解决**:
- 确认在hr.module.ts中注册了Controller
- 确认应用已重启

### 问题2: 字段没有dataSource

**检查**:
```sql
SELECT * FROM "CustomField"
WHERE code IN ('gender', 'nation', 'maritalStatus');
```

**解决**:
```bash
psql -U jy_admin -d jy_production -f scripts/fix-datasource-fields.sql
```

### 问题3: 选项为空数组

**检查**:
```sql
SELECT COUNT(*) FROM "DataSourceOption" o
INNER JOIN "DataSource" d ON d.id = o."dataSourceId"
WHERE d.code = 'GENDER';
```

**解决**: 检查种子数据是否已插入
