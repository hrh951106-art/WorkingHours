# 收卡范围逻辑测试指南

## 概述

本文档用于验证收卡范围逻辑的正确性，包括有排班和未排班日期的收卡范围计算。

## 业务规则

### 1. 有排班日期的收卡范围

**公式**：
```
收卡范围 = [排班开始时间 − 开始早范围, 排班结束时间 + 结束晚范围]
```

**示例**：
- 排班时间：08:00 - 17:00
- 开始早范围：120分钟（2小时）
- 结束晚范围：120分钟（2小时）
- 收卡范围：06:00 - 19:00

### 2. 未排班日期的收卡范围

#### 2.1 前后都有排班

**公式**：
```
收卡范围 = [前一天结束时间+1min, 后一天开始时间-1min]
```

**示例**：
- 前一天排班：08:00 - 17:00，收卡范围：06:00 - 19:00
- 后一天排班：08:00 - 17:00，收卡范围：06:00 - 19:00
- 当天（未排班）收卡范围：19:01 - 05:59（第二天）

**注意**：如果前后两天收卡范围有交叉，使用交叉范围的中间时间点进行成对归属。

#### 2.2 只有前一天有排班

**公式**：
```
收卡范围 = [前一天结束时间+1min, 23:59:59]
```

**示例**：
- 前一天排班：08:00 - 17:00，收卡范围：06:00 - 19:00
- 当天（未排班）收卡范围：19:01 - 23:59:59

#### 2.3 只有后一天有排班

**公式**：
```
收卡范围 = [00:00:00, 后一天开始时间-1min]
```

**示例**：
- 后一天排班：08:00 - 17:00，收卡范围：06:00 - 19:00
- 当天（未排班）收卡范围：00:00:00 - 05:59

#### 2.4 前后都无排班

**公式**：
```
收卡范围 = [00:00:00, 23:59:59]
```

## 收卡范围交叉处理

### 规则

当未排班日期的收卡范围与前后排班日期的收卡范围有交叉时：

1. 计算交叉范围
2. 找到交叉范围的中间时间点
3. 根据中间时间点将打卡记录成对
4. 按照时间点归属到对应的日期

### 示例

**场景**：
- 前一天排班：22:00 - 06:00（夜班），收卡范围：20:00 - 08:00
- 当天（未排班）
- 后一天排班：08:00 - 17:00，收卡范围：06:00 - 19:00

**交叉范围**：
- 前一天收卡结束：08:00
- 后一天收卡开始：06:00
- 交叉范围：06:00 - 08:00
- 中间时间点：07:00

**归属规则**：
- 07:00 之前的打卡 → 归属到前一天
- 07:00 之后的打卡 → 归属到当天（未排班）

## 测试场景

### 场景1：有排班日期

**输入**：
- 排班日期：2025-01-15
- 班次时间：08:00 - 17:00
- 开始早范围：120分钟
- 结束晚范围：120分钟

**预期收卡范围**：
- 开始时间：2025-01-15 06:00:00
- 结束时间：2025-01-15 19:00:00
- 范围类型：SCHEDULED

**验证SQL**：
```sql
-- 检查摆卡记录的收卡时间
SELECT
  employeeNo,
  pairDate,
  inPunchTime,
  outPunchTime,
  shiftName
FROM PunchPair
WHERE employeeNo = 'TEST001'
  AND pairDate = '2025-01-15'
ORDER BY inPunchTime;
```

### 场景2：未排班日期 - 前后都有排班

**输入**：
- 前一天排班：2025-01-14，08:00 - 17:00
- 当天未排班：2025-01-15
- 后一天排班：2025-01-16，08:00 - 17:00
- 开始早范围：120分钟
- 结束晚范围：120分钟

**预期收卡范围**：
- 前一天收卡结束：2025-01-14 19:00:00
- 后一天收卡开始：2025-01-16 06:00:00
- 当天收卡范围：2025-01-14 19:01:00 - 2025-01-16 05:59:00
- 范围类型：COMBINED

**验证日志**：
```
[收卡范围] 员工: TEST001, 日期: 2025-01-15, 范围: 2025-01-14T19:01:00.000Z - 2025-01-16T05:59:00.000Z, 类型: COMBINED
```

### 场景3：未排班日期 - 只有前一天有排班

**输入**：
- 前一天排班：2025-01-14，08:00 - 17:00
- 当天未排班：2025-01-15
- 后一天无排班

**预期收卡范围**：
- 前一天收卡结束：2025-01-14 19:00:00
- 当天收卡范围：2025-01-14 19:01:00 - 2025-01-15 23:59:59
- 范围类型：PREVIOUS_DAY

### 场景4：未排班日期 - 只有后一天有排班

**输入**：
- 前一天无排班
- 当天未排班：2025-01-15
- 后一天排班：2025-01-16，08:00 - 17:00

**预期收卡范围**：
- 后一天收卡开始：2025-01-16 06:00:00
- 当天收卡范围：2025-01-15 00:00:00 - 2025-01-16 05:59:00
- 范围类型：NEXT_DAY

### 场景5：未排班日期 - 前后都无排班

**输入**：
- 前一天无排班
- 当天未排班：2025-01-15
- 后一天无排班

**预期收卡范围**：
- 当天收卡范围：2025-01-15 00:00:00 - 2025-01-15 23:59:59
- 范围类型：FULL_DAY

### 场景6：收卡范围交叉

**输入**：
- 前一天排班：2025-01-14，22:00 - 06:00（夜班）
- 当天未排班：2025-01-15
- 后一天排班：2025-01-16，08:00 - 17:00
- 开始早范围：120分钟
- 结束晚范围：120分钟

**计算**：
- 前一天收卡范围：2025-01-14 20:00 - 2025-01-15 08:00
- 后一天收卡范围：2025-01-16 06:00 - 2025-01-16 19:00
- 交叉范围：2025-01-15 06:00 - 2025-01-15 08:00
- 中间时间点：2025-01-15 07:00

**打卡归属**：
- 2025-01-15 06:30 的打卡 → 归属到前一天（2025-01-14）
- 2025-01-15 07:30 的打卡 → 归属到当天（2025-01-15）

## 注意事项

### 1. 排班日期的查找

- 查找前一天排班：`scheduleDate < 目标日期 00:00:00`
- 查找后一天排班：`scheduleDate > 目标日期 23:59:59`
- 按日期降序/升序排序，取第一条

### 2. 收卡范围的边界

- 前一天结束+1分钟：避免重复
- 后一天开始-1分钟：避免重复
- 全天范围：00:00:00 - 23:59:59

### 3. 夜班处理

- 跨夜班次的 startDate/endDate 字段：
  - `+0`：当天
  - `+1`：第二天
- 计算收卡范围时需要考虑跨夜情况

### 4. 摆卡记录的shiftId

- 有排班日期：shiftId 为实际班次ID
- 未排班日期：shiftId 为 null
- shiftName 为 "未排班"

### 5. 工时计算触发

- 无论是有排班还是未排班日期，都需要触发工时计算
- 未排班日期的摆卡记录同样会触发 calculateFromPunchPair

## 已知问题和限制

### 1. 收卡范围交叉的成对归属

当前实现了基本的成对归属逻辑，但可能需要进一步优化：
- 复杂场景下的归属规则
- 跨天打卡的归属判断

### 2. 性能优化

- 未排班日期需要查询前后排班，可能影响性能
- 可以考虑缓存排班信息
- 批量处理时可以预先加载所有相关排班

### 3. 边界情况

- 排班信息缺失的处理
- 收卡范围无效的情况（如开始时间 > 结束时间）
- 跨时区的处理
