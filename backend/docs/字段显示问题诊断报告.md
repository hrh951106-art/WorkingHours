## 🚨 发现的问题

### 问题1: 保存失败 - gender字段必填但未提供

**错误信息**:
```
PrismaClientValidationError: Invalid `this.prisma.employee.create()` invocation
Argument `gender` is missing.
```

**原因**:
- Employee表中`gender`字段是必填的（NOT NULL）
- 创建员工时，如果前端没有提供gender字段，或者gender为空，就会报错

**解决方案**:
1. 方案A: 修改数据库schema，将gender字段改为可空
2. 方案B: 在代码中为gender设置默认值
3. 方案C: 确保前端必填并发送gender字段

**推荐方案**: 方案C - 确保前端验证并发送gender字段

### 问题2: 基本信息页签字段显示问题

**配置情况**:
- 总字段数: 22
- 隐藏字段: 1（idCard）

**问题原因**:

3. **可能的前端渲染问题**: 前端可能无法正确处理某些字段类型

**重复字段详情**:
| 字段代码 | 字段名称 | 排序 | 是否重复 |
|---------|---------|------|---------|
| mobile | 手机号码 | 12 | ✅ 与phone重复 |
| phone | 手机号码 | 13 | ✅ 与mobile重复 |

**排序冲突**:
- sort=0: A05, emergencyContact（2个字段）
- sort=1: emergencyPhone, name（2个字段）
- sort=2: birthDate, emergencyRelation（2个字段）
- sort=3: gender, homeAddress（2个字段）
- sort=4: age, homePhone（2个字段）

```typescript
Object.keys(restData).forEach(key => {
  const value = restData[key];

  if (employeeFields.includes(key)) {
    // Employee 表字段
    let finalValue = value;

    // 修复: gender字段必填处理
    if (key === 'gender' && (!value || value === '')) {
      // 如果没有提供gender，使用默认值

    if (key === 'entryDate' || key === 'birthDate') {
      if (typeof value === 'string') {
        finalValue = new Date(value);
      }
    }

    employeeData[key] = finalValue;
  } else if (workInfoFields.includes(key)) {
    // ... 其余代码不变
  }
});
```

### 修复2: 统一字段排序

**执行SQL**:

```sql
-- 修复基本信息页签字段排序
UPDATE EmployeeInfoTabField
SET sort = CASE fieldCode
    WHEN 'A05' THEN 0
    WHEN 'emergencyContact' THEN 1
    WHEN 'emergencyPhone' THEN 2
    WHEN 'name' THEN 3
    WHEN 'birthDate' THEN 4
    WHEN 'emergencyRelation' THEN 5
    WHEN 'gender' THEN 6
    WHEN 'age' THEN 7
    WHEN 'nation' THEN 8
    WHEN 'maritalStatus' THEN 9
    WHEN 'politicalStatus' THEN 10
    WHEN 'nativePlace' THEN 11
    WHEN 'idCard' THEN 12
    WHEN 'A06' THEN 13
    WHEN 'householdRegister' THEN 14
    WHEN 'homeAddress' THEN 15
    WHEN 'homePhone' THEN 16
    WHEN 'currentAddress' THEN 17
    WHEN 'mobile' THEN 18
    WHEN 'phone' THEN 19
    WHEN 'email' THEN 20
    WHEN 'photo' THEN 21
    ELSE sort
END
WHERE tabId = (SELECT id FROM EmployeeInfoTab WHERE code = 'basic_info');
```
