## 数据库表结构概览

### 核心表关系

```
Employee (员工主表)
├── Organization (组织架构) - 多对一
├── WorkInfoHistory (工作信息历史) - 一对多
├── EmployeeEducation (学历信息) - 一对多
├── EmployeeWorkExperience (工作经历) - 一对多
├── EmployeeFamilyMember (家庭成员) - 一对多
└── CustomField (自定义字段定义) - 多对多

数据源关系:
DataSource (数据源主表)
├── DataSourceOption (数据源选项) - 一对多
└── CustomField (自定义字段) - 一对多

页签配置关系:
EmployeeInfoTab (员工信息页签)
├── EmployeeInfoTabGroup (页签分组) - 一对多
└── EmployeeInfoTabField (页签字段) - 一对多
```

## 生产环境问题排查

### 问题描述

**本地环境**: 系统内置字段可以选择到下拉值
**生产环境**: 数据源有，但是选择不到值

### 排查步骤

#### 第一步: 运行诊断SQL

```bash
psql -U username -d database_name -f scripts/query-all-fields-and-datasources.sql
```

#### 第二步: 检查关键项

##### 2.1 检查DataSource表是否有数据

```sql
SELECT
    id,
    code,
    name,
    type,
    "isSystem",
    status
FROM "DataSource"
WHERE "isSystem" = true
ORDER BY sort;
```

**预期结果**: 应该有11个系统内置数据源

##### 2.2 检查DataSourceOption表是否有数据

```sql
SELECT
    d.code AS "数据源代码",
    d.name AS "数据源名称",
    COUNT(o.id) AS "选项数量"
FROM "DataSource" d
LEFT JOIN "DataSourceOption" o ON o."dataSourceId" = d.id
WHERE d."isSystem" = true
GROUP BY d.id, d.code, d.name
ORDER BY d.sort;
```

**预期结果**: 每个数据源应该有至少1个选项

##### 2.3 检查CustomField是否关联了数据源

```sql
SELECT
    cf.code AS "字段代码",
    cf.name AS "字段名称",
    cf.type AS "字段类型",
    cf."dataSourceId" AS "数据源ID",
    d.code AS "数据源代码",
    d.name AS "数据源名称"
FROM "CustomField" cf
LEFT JOIN "DataSource" d ON d.id = cf."dataSourceId"
WHERE cf.code IN ('gender', 'marital_status', 'political_status')
ORDER BY cf.code;
```

**预期结果**: 每个字段应该关联到对应的数据源

##### 2.4 检查EmployeeInfoTabField的fieldType

```sql
SELECT
    f.fieldCode AS "字段代码",
    f.fieldName AS "字段名称",
    f.fieldType AS "字段类型",
    t.code AS "页签代码"
FROM "EmployeeInfoTabField" f
INNER JOIN "EmployeeInfoTab" t ON t.id = f."tabId"
WHERE f.fieldCode IN ('gender', 'maritalStatus', 'politicalStatus', 'educationLevel')
ORDER BY t.code, f.fieldCode;
```

**预期结果**: fieldType应该是 `SELECT` 或 `CUSTOM`

### 常见问题及解决方案

#### 问题1: CustomField未关联数据源

**症状**: 字段类型是SELECT，但CustomField.dataSourceId为NULL

**解决方案**:
```sql
-- 更新gender字段
UPDATE "CustomField"
SET "dataSourceId" = (SELECT id FROM "DataSource" WHERE code = 'gender')
WHERE code = 'gender';

-- 更新maritalStatus字段
UPDATE "CustomField"
SET "dataSourceId" = (SELECT id FROM "DataSource" WHERE code = 'marital_status')
WHERE code = 'marital_status';
```

#### 问题2: EmployeeInfoTabField的fieldType不正确

**症状**: 下拉字段的fieldType是SYSTEM而不是SELECT

**解决方案**:
```sql
UPDATE "EmployeeInfoTabField"
SET fieldType = 'SELECT'
WHERE fieldCode IN (
    'gender', 'nation', 'maritalStatus', 'politicalStatus',
    'educationLevel', 'employeeType', 'position', 'jobLevel'
)
AND fieldType != 'SELECT';
```

#### 问题3: 字段命名不一致

**症状**: CustomField使用下划线命名(snake_case)，EmployeeInfoTabField使用驼峰命名(camelCase)

**解决方案**:
```sql
-- 统一为驼峰命名
UPDATE "CustomField"
SET code = CASE
    WHEN code = 'marital_status' THEN 'maritalStatus'
    WHEN code = 'political_status' THEN 'politicalStatus'
    WHEN code = 'education_level' THEN 'educationLevel'
    WHEN code = 'employee_type' THEN 'employeeType'
    WHEN code = 'job_level' THEN 'jobLevel'
    ELSE code
END
WHERE code IN ('marital_status', 'political_status', 'education_level', 'employee_type', 'job_level');
```

#### 问题4: 数据源选项的isActive字段为false

**症状**: 数据源和选项都存在，但前端看不到选项

**解决方案**:
```sql
UPDATE "DataSourceOption"
SET "isActive" = true
WHERE "isActive" = false;
```

### A. 字段命名规范

- **数据库表字段**: 驼峰命名 (camelCase) - 例: `employeeNo`, `orgId`
- **页签字段代码**: 驼峰命名 (camelCase) - 例: `maritalStatus`, `politicalStatus`
- **CustomField代码**: 驼峰命名 (camelCase) - 例: `maritalStatus`, `politicalStatus`
- **数据源代码**: 下划线命名 (snake_case) - 例: `marital_status`, `political_status`
