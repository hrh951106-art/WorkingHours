# 新增第二个层级错误修复报告

## 问题描述
用户在新增第二个层级时，前端提示"服务器内部错误"

## 错误日志
```
Unique constraint failed on the fields: (`code`)
Invalid `tx.accountHierarchyConfig.create()` invocation
```

## 根本原因分析

### 问题所在
在 `src/modules/account/account.service.ts` 的 `batchSaveHierarchyLevels` 方法中：

```typescript
// 使用事务确保数据一致性
return this.prisma.$transaction(async (tx) => {
  // 先删除所有现有配置
  await tx.accountHierarchyLevelDetail.deleteMany({});
  await tx.accountHierarchyConfig.deleteMany({});

  // 批量创建新配置
  const createdConfigs = await Promise.all(
    levels.map(async (level, index) => {
      // ❌ 问题：每次都在事务内查询existingCodes
      const existingCodes = await tx.accountHierarchyConfig.findMany({
        select: { code: true },
        where: { status: 'ACTIVE' },
      });
      const codes = existingCodes.map(c => c.code);

      // ❌ 问题：因为刚删除了所有配置，existingCodes永远返回空数组
      const code = StringUtils.generateSequentialCode('AHC', codes);

      const config = await tx.accountHierarchyConfig.create({...});
    })
  );
});
```

### 执行流程
1. **第一次循环**（创建第一个层级）：
   - 查询 existingCodes → 返回 `[]`（因为刚删除了所有数据）
   - 生成 code → `AHC001`
   - 创建成功 ✅

2. **第二次循环**（创建第二个层级）：
   - 查询 existingCodes → 返回 `[]`（因为第一个层级还没提交到数据库）
   - 生成 code → `AHC001`（又生成了相同的code）
   - 创建失败 ❌ → 违反唯一约束

## 修复方案

### 修改后的代码
```typescript
// 使用事务确保数据一致性
return this.prisma.$transaction(async (tx) => {
  // 先删除所有现有配置
  await tx.accountHierarchyLevelDetail.deleteMany({});
  await tx.accountHierarchyConfig.deleteMany({});

  // ✅ 修复：在循环外预先生成所有唯一codes
  const { StringUtils } = require('../../common/utils');
  const generatedCodes: string[] = [];

  // 批量创建新配置
  const createdConfigs = await Promise.all(
    levels.map(async (level, index) => {
      // ✅ 修复：使用已生成的codes作为参考
      const code = StringUtils.generateSequentialCode('AHC', generatedCodes);
      generatedCodes.push(code); // 将新code加入数组

      const config = await tx.accountHierarchyConfig.create({
        data: {
          code,
          name: level.name,
          mappingType: level.mappingType,
          mappingValue: level.mappingValue,
          dataSourceId: level.dataSourceId,
          level: index + 1,
          sort: level.sort || 0,
          status: level.status || 'ACTIVE',
        },
      });

      // 同步层级明细
      await this.syncHierarchyLevelDetails(tx, config);

      return config;
    })
  );

  return createdConfigs;
});
```

### 修复原理
- **在循环外**创建 `generatedCodes` 数组
- 每次生成code时，将新code加入数组
- 下一次生成时，已生成的codes会被考虑在内
- 保证所有code都是唯一的
