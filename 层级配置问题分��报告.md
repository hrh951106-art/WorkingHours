# 劳动力账户 - 层级配置页签无法新增层级问题分析

## 问题描述
用户在"劳动力账户管理 > 层级配置"页签中无法新增层级

## 问题原因分析

经过代码分析，发现可能存在以下几个问题：

### 1. 按钮显示逻辑问题
在 `frontend/src/pages/account/AccountPage.tsx` 第194行，新增层级按钮只在表格最后一行显示：

```tsx
{isLastRow && (
  <Button
    type="primary"
    size="small"
    icon={<PlusOutlined />}
    onClick={handleAddLevel}
    disabled={isMaxLevel}
  />
)}
```

### 2. React Query 缓存更新问题
在 `handleAddLevel` 函数中（第114行），使用了 `setQueryData` 更新缓存：

```tsx
queryClient.setQueryData(['hierarchyConfig'], (old: any[]) => [...(old || []), newLevel]);
```

但可能存在以下问题：
- 缓存更新后，组件没有正确重新渲染
- 或者在某些情况下，查询被重新获取，覆盖了本地修改

### 3. 可能的竞态条件
如果用户快速点击按钮，或者在数据加载过程中点击，可能导致状态不一致。

## 修复方案

### 方案1：使用乐观更新 + 失效查询
```tsx
const handleAddLevel = () => {
  if (hierarchyConfig && hierarchyConfig.length >= 15) {
    message.warning('最多支持15个层级，已达到上限');
    return;
  }

  const newLevel = {
    id: `temp_${Date.now()}`,
    name: '',
    mappingType: undefined,
    mappingValue: undefined,
    sort: (hierarchyConfig?.length || 0),
    status: 'ACTIVE',
  };

  // 使用乐观更新
  queryClient.setQueryData(['hierarchyConfig'], (old: any[]) => [...(old || []), newLevel]);

  // 可选：立即失效查询以确保数据同步
  // queryClient.invalidateQueries({ queryKey: ['hierarchyConfig'] });
};
```

### 方案2：添加调试日志
在 `handleAddLevel` 函数中添加调试日志：

```tsx
const handleAddLevel = () => {
  console.log('handleAddLevel called');
  console.log('Current hierarchyConfig:', hierarchyConfig);

  if (hierarchyConfig && hierarchyConfig.length >= 15) {
    message.warning('最多支持15个层级，已达到上限');
    return;
  }

  const newLevel = {
    id: `temp_${Date.now()}`,
    name: '',
    mappingType: undefined,
    mappingValue: undefined,
    sort: (hierarchyConfig?.length || 0),
    status: 'ACTIVE',
  };

  console.log('Adding new level:', newLevel);

  queryClient.setQueryData(['hierarchyConfig'], (old: any[]) => {
    const newData = [...(old || []), newLevel];
    console.log('Updated data:', newData);
    return newData;
  });
};
```

### 方案3：检查按钮渲染条件
确保按钮在正确的条件下渲染。可以修改为始终显示新增按钮：

```tsx
<Button
  type="primary"
  size="small"
  icon={<PlusOutlined />}
  onClick={handleAddLevel}
  disabled={isMaxLevel}
  title="新增层级"
/>
```

## 诊断步骤

1. 打开浏览器开发者工具（F12）
2. 切换到 Console 标签页
3. 访问"账户管理 > 层级配置"页面
4. 点击"新增层级"按钮
5. 检查控制台是否有错误或警告
6. 检查是否有日志输出
7. 在 Network 标签页检查API请求

## 检查清单

- [ ] 检查浏览器控制台是否有JavaScript错误
- [ ] 检查Network标签页，确认API请求是否成功
- [ ] 检查是否有权限相关的错误
- [ ] 检查React Query DevTools中的缓存状态
- [ ] 检查是否已达到15个层级的上限

## 需要的更多信息

为了准确诊断问题，需要了解：

1. **具体表现**：
   - 点击按钮后没有任何反应？
   - 还是按钮被隐藏或禁用了？
   - 还是点击后出现错误提示？

2. **当前状态**：
   - 已经有多少个层级配置？
   - 是否是第一次配置（0个层级）？
   - 还是在已有层级的基础上添加？

3. **错误信息**：
   - 浏览器控制台是否有错误？
   - 是否有弹窗提示？

## 建议的修复

基于最佳实践，建议修改 `AccountPage.tsx`：

1. 在表格上方添加一个固定的"新增层级"按钮
2. 保持表格最后一行的按钮作为快捷方式
3. 添加详细的错误处理和用户反馈

这样可以确保用户在任何情况下都能找到新增按钮。
