# 流程参与人配置功能使用说明

## 功能概述

现在工作流系统支持独立的参与人配置管理。您可以预先定义好常用的参与人组（如"部门经理"、"车间主任"、"HR主管"等），然后在设计工作流审批节点时，直接勾选这些预定义的参与人配置即可，无需每次都重新配置。

---

## 一、参与人配置管理

### 1. 访问参与人配置页面

**路径**: 工作流管理 → 参与人配置

**URL**: `/workflow/participants`

### 2. 创建参与人配置

点击"新增配置"按钮，填写以下信息：

#### 基本信息
- **配置代码**: 全局唯一标识，只能包含大写字母、数字和下划线
  - 示例: `DEPT_MANAGER`、`WORKSHOP_DIRECTOR`、`HR_SUPERVISOR`
  - 保存后不可修改

- **配置名称**: 显示名称
  - 示例: "部门经理"、"车间主任"、"HR主管"

- **配置类型**:
  - `FIXED_USER` - 固定人员
  - `ORG_MANAGER` - 组织管理者
  - `SUBMITTER` - 提交人
  - `ORG_LEADER` - 组织领导

- **排序号**: 用于控制列表显示顺序

#### 参与人配置

点击"配置参与人"按钮，在弹窗中添加参与人：

**固定人员类型**:
- 从用户列表中选择具体的人员
- 支持搜索（按姓名或员工编号）

**组织管理者类型**:
- 选择组织层级：
  - 申请人
  - 所在组织
  - 所在组织向上一级
  - 所在组织向上2级
  - 所在组织向上3级
  - 所在组织向上4级
  - 所在组织向上5级

#### 描述（可选）
添加配置的说明文字，方便其他用户理解。

### 3. 管理参与人配置

列表页提供以下操作：
- **编辑**: 修改配置名称、类型、参与人、描述、排序
- **删除**: 删除配置（需要确认）
- **启用/停用**: 点击状态标签切换

---

## 二、在工作流设计器中使用参与人配置

### 1. 打开工作流设计器

**路径**: 工作流定义 → 新增流程（或编辑现有流程）→ 点击"设计"按钮

### 2. 添加审批/抄送节点

- 点击工具栏的"审批"按钮添加审批节点
- 点击"抄送"按钮添加抄送节点

### 3. 配置审批人/抄送人

1. 点击画布上的审批/抄送节点
2. 右侧配置面板会展开
3. **审批节点**:
   - 先选择"审批方式"（串行/并行/或签）
   - 在"审批人配置"区域，勾选需要的参与人配置
   - 可以多选

4. **抄送节点**:
   - 在"抄送人配置"区域，勾选需要的参与人配置
   - 可以多选

### 4. 保存配置

点击"保存配置"按钮保存节点设置。

## 三、使用示例

### 示例1：请假审批流程

假设您的组织有以下层级：
```
公司
  └─ 部门
      └─ 科室
```

**预定义参与人配置**:
1. `DIRECT_SUPERVISOR` - 直属上级（所在组织向上一级）
2. `DEPT_MANAGER` - 部门经理（所在组织向上2级）
3. `HR_ADMIN` - HR管理员（固定人员）

**流程设计**:
1. 开始 → 审批节点1（直属上级）→ 审批节点2（部门经理）→ 结束
2. 审批节点1配置：勾选 `DIRECT_SUPERVISOR`
3. 审批节点2配置：勾选 `DEPT_MANAGER`

### 示例2：报工审批流程

**预定义参与人配置**:
1. `WORKSHOP_DIRECTOR` - 车间主任（所在组织向上一级）
2. `PRODUCTION_MANAGER` - 生产经理（所在组织向上2级）
3. `FINANCE_APPROVER` - 财务审批人（固定人员）

**流程设计**:
1. 开始 → 审批节点1（车间主任）→ 审批节点2（生产经理）→ 抄送（财务）→ 结束
2. 审批节点1配置：勾选 `WORKSHOP_DIRECTOR`
3. 审批节点2配置：勾选 `PRODUCTION_MANAGER`
4. 抄送节点配置：勾选 `FINANCE_APPROVER`

## 五、优势

### 1. 复用性
- 一次配置，多处使用
- 不同工作流可以共享同一参与人配置

### 2. 易维护
- 参与人变更时，只需修改配置，无需修改每个工作流
- 例如：部门经理更换时，更新对应的参与人配置即可

### 3. 标准化
- 统一的命名规范
- 统一的配置标准
- 降低错误配置的可能性

### 4. 效率提升
- 设计工作流时快速选择，无需逐个添加参与人
- 特别适用于复杂的多级审批流程

---

## 六、注意事项

1. **代码唯一性**: 配置代码必须全局唯一，创建后不可修改
2. **参与人数量**: 每个配置可以包含多个参与人
3. **类型混合**: 可以在同一个节点中同时选择多个参与人配置
4. **状态控制**: 停用的配置不会在选择列表中显示
5. **权限管理**: 请确保用户有权限访问参与人配置管理功能

## 七、API接口

### 后端接口

- `GET /workflow/participants` - 获取参与人配置列表
- `GET /workflow/participants/:id` - 获取配置详情
- `POST /workflow/participants` - 创建参与人配置
- `PUT /workflow/participants/:id` - 更新参与人配置
- `PUT /workflow/participants/:id/status` - 更新配置状态
- `DELETE /workflow/participants/:id` - 删除参与人配置

### 前端路由

- `/workflow/participants` - 参与人配置列表
- `/workflow/participants/create` - 新增参与人配置
- `/workflow/participants/:id/edit` - 编辑参与人配置

## 八、常见问题

### Q1: 修改参与人配置后，已经运行的工作流会受影响吗？
A: 不会。已运行的工作流使用的是创建时的快照数据。

### Q2: 可以在同一个审批节点中使用多个参与人配置吗？
A: 可以。系统会汇总所有选中配置的参与人。

### Q3: 固定人员离职后怎么办？
A: 需要编辑对应的参与人配置，移除离职人员，添加新的接替者。

### Q4: 参与人配置可以嵌套使用吗？
A: 目前不支持嵌套。每个参与人配置都是独立的。
