# BI报表工具 - 数据模型模块完整开发

## 更新时间
2025-05-14

## 模块概述

数据模型模块是BI报表工具的核心功能，用于将原始数据库表抽象为可配置的数据模型，支持维度和度量的定义，为后续的报表设计提供数据基础。

### 3. 智能字段识别 ✅

**维度字段自动识别**:
- 字符串类型 → 维度
- 日期类型 → 维度
- 布尔类型 → 维度
- 表元数据标注为dimension → 维度

**度量字段自动识别**:
- 数字类型 + 无元数据标注 → 度量
- 表元数据标注为measure → 度量
- 自动设置默认聚合函数为sum

#### 字段导入逻辑
```typescript
// 智能字段类型识别
const newFields: ModelField[] = availableFields.map((field) => {
  // 优先使用元数据中的字段类型
  let type: 'dimension' | 'measure' = 'dimension';
  if (field.fieldType === 'measure' ||
      (field.dataType === 'number' && !field.fieldType)) {
    type = 'measure';
  }

  return {
    name: field.name,              // 中文名称
    code: field.code,              // 英文代码
    type,                          // 维度/度量
    dataType: field.dataType,      // 数据类型
    sourceType: 'column',
    sourceExpr: field.code,
    aggregation: type === 'measure' ? 'sum' : undefined,
    description: field.description, // 字段描述
    isVisible: true,
    isSearchable: false,
  };
});
```

## 用户使用流程

### 创建数据模型

1. **进入模型管理**
   - 访问路径：`/bi-report/models`
   - 点击"创建模型"按钮

2. **填写基本信息**
   - 输入模型名称（如：员工工时分析模型）
   - 输入模型代码（如：EMPLOYEE_WORK_HOURS）
   - 选择模型类型（默认：数据表）
   - 选择源表（支持搜索，显示中文表名）
   - 查看源表信息（描述、记录数）

3. **配置字段**
   - 方式一：从表导入
     - 点击"从表导入"按钮
     - 在Transfer组件中选择需要的字段
     - 查看字段的中文信息
     - 点击确定导入
   - 方式二：全部导入
     - 点击"全部导入"按钮
     - 系统自动导入所有字段
     - 智能识别维度/度量
   - 方式三：手动添加
     - 点击"手动添加"按钮
     - 填写字段详细信息
     - 保存字段

4. **预览数据**
   - 切换到"数据预览"标签
   - 点击"预览数据"按钮
   - 查看查询结果（前20条）

5. **保存模型**
   - 点击"保存"按钮
   - 系统验证配置
   - 保存成功后返回列表

### 编辑数据模型

1. 在列表中点击"编辑"按钮
2. 修改基本信息或字段配置
3. 预览数据验证
4. 保存更改

### 删除数据模型

1. 在列表中点击"删除"按钮
2. 确认删除操作
3. 系统删除模型及相关配置

## API示例

### 创建模型
```bash
POST /bi-report/models
Content-Type: application/json

{
  "name": "员工工时分析模型",
  "code": "EMPLOYEE_WORK_HOURS",
  "type": "table",
  "sourceTable": "WorkHourResult",
  "description": "用于分析员工每日工时、加班工时等数据指标",
  "fields": [
    {
      "name": "员工姓名",
      "code": "employeeName",
      "type": "dimension",
      "dataType": "string",
      "sourceType": "column",
      "sourceExpr": "employeeName",
      "isVisible": true,
      "isSearchable": true
    },
    {
      "name": "工作工时",
      "code": "workHours",
      "type": "measure",
      "dataType": "number",
      "aggregation": "sum",
      "sourceType": "column",
      "sourceExpr": "workHours",
      "isVisible": true
    }
  ]
}
```

### 查询模型数据
```bash
POST /bi-report/models/1/query
Content-Type: application/json

{
  "dimensions": ["employeeName", "workDate"],
  "measures": [
    { "field": "workHours", "aggregation": "sum" },
    { "field": "overtimeHours", "aggregation": "sum" }
  ],
  "filters": {
    "workDate": {
      "operator": ">=",
      "value": "2025-05-01"
    }
  },
  "groupBy": ["employeeName", "workDate"],
  "orderBy": [{ "field": "workDate", "direction": "DESC" }],
  "limit": 20
}
```
