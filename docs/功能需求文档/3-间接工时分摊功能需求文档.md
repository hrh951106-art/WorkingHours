# 间接工时分摊功能需求文档

## 一、功能概述

间接工时分摊功能用于配置间接工时（如管理人员工资、办公费用等）的分摊规则，将间接成本按照设定的维度和比例分摊到直接生产部门或产品上。该功能通过向导式配置界面，支持复杂的数据源过滤、分摊维度定义和分摊规则设置。

## 二、页面路径
- 前端路由：`/allocation/config`
- 页面组件：`frontend/src/pages/allocation/AllocationConfigPage.tsx`
- 页签：间接工时

## 四、数据字段说明

### 4.1 AllocationConfig（分摊配置基本信息）

| 字段名 | 类型 | 必填 | 说明 | 验证规则 |
|--------|------|------|------|----------|
| id | number | 是 | 配置ID（自动生成） | - |
| configCode | string | 是 | 配置编码 | 唯一，大写字母+数字 |
| configName | string | 是 | 配置名称 | 长度2-100字符 |
| configType | string | 是 | 配置类型 | INDIRECT_HOURS（间接工时） |
| version | number | 是 | 配置版本 | 默认1，每次编辑递增 |
| status | string | 是 | 配置状态 | DRAFT/ACTIVE/INACTIVE |
| effectiveDate | string | 否 | 生效日期 | YYYY-MM-DD格式 |
| expiryDate | string | 否 | 失效日期 | YYYY-MM-DD格式 |
| description | string | 否 | 配置描述 | 最多500字符 |
| createdBy | string | 是 | 创建人 | 自动获取当前用户 |
| updatedBy | string | 是 | 更新人 | 自动获取当前用户 |
| createdAt | string | 是 | 创建时间 | 自动生成 |
| updatedAt | string | 是 | 更新时间 | 自动生成 |

### 4.2 AllocationSourceConfig（数据源配置）

| 字段名 | 类型 | 必填 | 说明 | 验证规则 |
|--------|------|------|------|----------|
| sourceType | string | 是 | 源数据类型 | EMPLOYEE_HOURS（员工工时） |
| employeeFilter | object | 是 | 员工过滤条件 | 见下文详细说明 |
| accountFilter | object | 是 | 账户过滤条件 | 见下文详细说明 |
| attendanceCodes | array | 否 | 出勤代码列表 | 出勤代码ID数组 |

#### 4.2.1 EmployeeFilter（员工过滤器）

```typescript
interface EmployeeFilter {
  fieldGroups: FieldGroup[];  // 字段组列表
}

interface FieldGroup {
  groupId: number;           // 字段组ID
  groupName: string;         // 字段组名称
  logicalOperator: 'AND' | 'OR';  // 逻辑关系
  conditions: Condition[];   // 条件列表
}

interface Condition {
  fieldId: number;           // 字段ID
  fieldName: string;         // 字段名称
  operator: 'EQUAL' | 'NOT_EQUAL' | 'IN' | 'NOT_IN' | 'LIKE' | 'BETWEEN';  // 操作符
  value: any;               // 字段值
}
```

#### 4.2.2 AccountFilter（账户过滤器）

```typescript
interface AccountFilter {
  hierarchySelections: HierarchySelection[];  // 层级选择列表
}

interface HierarchySelection {
  dimensionId: number;      // 维度ID（LINE/WORKSHOP/TEAM等）
  dimensionType: string;    // 维度类型
  level: number;            // 层级
  operator: 'INCLUDE' | 'EXCLUDE';  // 包含/排除
  selections: Selection[];  // 选择项列表
}

interface Selection {
  id: number;               // 选项ID
  name: string;             // 选项名称
  code: string;             // 选项编码
}
```

### 4.3 AllocationRuleConfig（分摊规则配置）

```typescript
interface AllocationRule {
  id?: number;              // 规则ID（编辑时存在）
  allocationScopeId?: number;  // 分摊范围ID（可选）
  allocationBasis: string;  // 分摊依据类型
  targets: AllocationTarget[];  // 分摊目标列表（固定为空数组）
}
```

#### 4.3.1 AllocationBasis（分摊依据类型）

可选值：
- `ACTUAL_HOURS`：实际工时
- `STD_HOURS`：标准工时
- `ACTUAL_YIELDS`：实际产量
- `EQUIVALENT_YIELDS`：当量产量
- `PRODUCT_STD_HOURS`：产品标准工时
- `EMPLOYEE_COUNT`：人员数量
- `FIXED_VALUE`：固定值

## 五、操作流程

### 5.1 新建间接工时分摊配置（向导式）

**流程步骤**：

#### 第一步：定义数据源
1. 点击"新建配置"按钮
2. 打开新建向导对话框，显示第一步：定义数据源
3. 配置基本信息：
   - 输入配置编码（必填，唯一）
   - 输入配置名称（必填）
   - 选择生效日期（必填）
   - 选择失效日期（可选）
   - 输入配置描述（可选）
4. 配置员工过滤条件：
   - 点击"添加过滤条件组"
   - 选择字段组（如"基本信息"、"工作信息"）
   - 选择字段（如"工号"、"姓名"、"部门"）
   - 选择操作符（等于、不等于、包含、不包含等）
   - 输入字段值
   - 设置条件组的逻辑关系（AND/OR）
   - 支持添加多个条件组
5. 配置账户过滤条件：
   - 点击"添加层级选择"
   - 选择维度类型（产线、车间、班组、工作中心、账户、成本中心）
   - 选择层级
   - 选择操作类型（包含/排除）
   - 选择具体选项
   - 支持添加多个层级选择
6. 配置出勤代码：
   - 选择要包含的出勤代码
   - 支持多选
7. 点击"下一步"按钮

#### 第二步：配置分摊规则
1. 显示分摊规则配置页面
2. 点击"添加规则"
3. 配置规则参数：
   - 选择分摊依据类型（必填）
   - 配置层级设置（可选）
4. 支持添加多条规则
5. 点击"上一步"返回第一步
6. 点击"完成"保存配置
7. 系统验证数据完整性
8. 验证通过后，调用API创建配置
9. 创建成功后，关闭对话框，刷新列表
10. 显示成功提示消息

**关键代码实现**：

## 六、业务规则

### 6.1 配置编码规则
- 配置编码必须唯一
- 只能包含大写字母、数字和下划线
- 长度2-50字符
- 建议格式：IH_YYYYMMDD_XXX（间接工时_日期_序号）

### 6.2 状态管理
- **DRAFT（草稿）**：新建配置的初始状态，可以编辑、删除
- **ACTIVE（已启用）**：配置生效，参与分摊计算，不可编辑和删除
- **INACTIVE（已停用）**：配置停用，不参与分摊计算，可以重新激活或删除

### 6.3 版本控制
- 每次编辑配置，版本号自动递增
- 版本号格式：主版本号.次版本号（如1.0, 1.1, 2.0）
- 修改基本信息、数据源、规则都会导致次版本号递增
- 重要变更可能导致主版本号递增

### 6.4 生效和失效日期
- 生效日期不能早于今天
- 失效日期必须晚于生效日期
- 如果当前日期在生效和失效日期之间，配置才有效
- 失效日期可选，如果不设置，配置永久有效

### 6.5 数据完整性
- 删除配置前，需要检查是否有关联的分摊结果
- 如果有关联数据，应提示用户无法删除
- 配置激活后，不允许修改数据源配置

### 6.6 分摊规则逻辑
- 规则按顺序执行
- targets字段固定为空数组，不使用
- 分摊依据决定如何计算分摊比例
- 如果配置了多条规则，按优先级执行

## 八、系统配置依赖

### 8.1 字段组配置
- 需要预先配置员工信息字段组
- 字段组来源：员工信息配置 > 信息页签
- 包括：基本信息、工作信息、自定义字段组

### 8.2 维度配置
- 需要预先配置分摊维度
- 维度类型：LINE、WORKSHOP、TEAM、WORK_CENTER、ACCOUNT、COST_CENTER
- 维度配置路径：系统配置 > 账户体系

### 8.3 出勤代码配置
- 需要预先配置出勤代码
- 出勤代码来源：考勤管理 > 出勤代码定义

### 8.4 字段数据源配置
- 字段选项需要配置数据源
- 数据源配置路径：系统配置 > 数据源配置

## 九、权限要求

### 9.1 功能权限
- 查看配置：`allocation:config:view`
- 新建配置：`allocation:config:create`
- 编辑配置：`allocation:config:edit`
- 激活配置：`allocation:config:activate`
- 停用配置：`allocation:config:deactivate`
- 删除配置：`allocation:config:delete`
- 复制配置：`allocation:config:copy`

### 9.2 数据权限
- 用户只能查看和操作自己创建的配置
- 管理员可以查看和操作所有配置

## 十、性能要求

### 10.1 查询性能
- 配置列表加载时间 < 2秒
- 支持分页或虚拟滚动
- 默认按创建时间倒序排列

### 10.2 保存性能
- 新建配置保存时间 < 3秒
- 编辑配置保存时间 < 3秒

### 10.3 数据验证
- 前端表单验证 < 1秒
- 后端业务逻辑验证 < 2秒

## 十一、异常处理

### 11.1 配置编码重复
- 创建或编辑时，如果配置编码已存在，提示："配置编码已存在，请使用其他编码"
- 阻止保存操作

### 11.2 数据验证异常
- 必填字段未填写时，高亮显示错误字段
- 显示具体的错误提示信息
- 日期逻辑错误（失效日期早于生效日期）时提示

### 11.3 业务逻辑异常
- 激活配置时，如果数据源配置不完整，提示："数据源配置不完整，无法激活"
- 删除配置时，如果已有关联数据，提示："该配置已产生分摊结果，无法删除"
- 编辑已启用配置时，提示："已启用的配置无法编辑，请先停用"

### 11.4 网络异常
- API调用失败时，显示错误提示
- 提供重试机制

## 十二、关键特性说明

### 12.1 向导式配置
- 分步骤引导用户完成配置
- 降低配置复杂度
- 支持上一步/下一步导航
- 每步数据独立验证

### 12.2 复杂过滤条件
- 支持多级过滤条件组合
- 支持AND/OR逻辑关系
- 支持多种操作符（等于、不等于、包含、不包含、介于等）
- 支持动态添加和删除条件

### 12.3 维度灵活配置
- 支持多种维度类型
- 支持包含/排除操作
- 支持多层级选择
- 可视化展示层级结构

### 12.4 版本控制
- 每次编辑自动递增版本号
- 保留配置历史记录
- 支持配置回滚（待实现）

## 十四、与其他功能的关系

### 14.1 与工时计算的关系
- 间接工时分摊依赖于工时计算结果
- 从工时计算结果中提取源工时数据
- 根据配置的过滤条件筛选源工时

### 14.2 与分摊计算的关系
- 间接工时分摊配置是分摊计算的输入
- 分摊计算根据配置规则执行分摊逻辑
- 生成分摊结果记录

### 14.3 与分摊结果查询的关系
- 配置激活后，产生分摊结果
- 分摊结果查询页面可以查看分摊结果
- 支持按配置批次号查询结果
