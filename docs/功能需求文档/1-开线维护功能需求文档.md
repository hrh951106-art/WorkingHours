# 开线维护功能需求文档

## 一、功能概述

开线维护功能用于管理生产线的开线记录，包括创建、编辑、删除、查询开线计划，以及支持批量操作和复制功能。该功能是工时分摊系统的基础，用于记录哪些生产线在什么时间段参与分摊。

## 二、页面路径
- 前端路由：`/allocation/line-maintenance`
- 页面组件：`frontend/src/pages/allocation/LineMaintenancePage.tsx`

### 3.2 查询条件区域
- **日期范围选择器**：用于筛选开线记录的日期范围
- **组织选择器**：树形选择器，支持选择特定组织
- **班次选择器**：下拉选择框，选择特定班次
- **查询按钮**：执行查询操作
- **重置按钮**：清空查询条件

### 3.3 操作按钮区域
- **新建按钮**：打开新建开线记录对话框
- **复制按钮**：复制选中的记录到新日期
- **批量删除按钮**：批量删除选中的记录
- **导出按钮**：导出查询结果（待实现）

### 3.4 数据表格区域
列定义：
1. **选择框**：支持单选和多选
2. **组织**：显示组织名称
3. **日期**：显示开线日期（格式：YYYY-MM-DD）
4. **班次**：显示班次名称
5. **开始时间**：显示开始时间（格式：HH:mm）
6. **结束时间**：显示结束时间（格式：HH:mm）
7. **状态**：显示记录状态（ACTIVE/INACTIVE）
8. **参与分摊**：是否参与工时分摊（是/否）
9. **操作**：编辑、删除按钮

## 四、数据字段说明

### 4.1 LineRecord（开线记录）

| 字段名 | 类型 | 必填 | 说明 | 验证规则 |
|--------|------|------|------|----------|
| id | number | 是 | 记录ID（自动生成） | - |
| orgId | number | 是 | 组织ID | 必须选择有效组织 |
| orgName | string | 是 | 组织名称 | 自动填充 |
| scheduleDate | string | 是 | 开线日期 | YYYY-MM-DD格式 |
| startTime | string | 是 | 开始时间 | HH:mm格式 |
| endTime | string | 是 | 结束时间 | HH:mm格式，必须晚于开始时间 |
| shiftId | number | 是 | 班次ID | 必须选择有效班次 |
| shiftName | string | 否 | 班次名称 | 自动填充 |
| participateInAllocation | boolean | 否 | 是否参与分摊 | 默认true |
| status | string | 是 | 状态 | ACTIVE或INACTIVE |
| createdAt | string | 是 | 创建时间 | 自动生成 |
| updatedAt | string | 是 | 更新时间 | 自动生成 |

### 4.2 Filter（查询条件）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| dateRange | [Dayjs, Dayjs] | 否 | 日期范围（开始日期，结束日期） |
| orgId | number | 否 | 组织ID |
| shiftId | number | 否 | 班次ID |

## 五、操作流程

### 5.1 查询开线记录

**流程步骤**：
1. 进入页面，系统自动加载所有开线记录
2. 用户可选择输入查询条件：
   - 选择日期范围
   - 选择组织
   - 选择班次
3. 点击"查询"按钮，系统根据条件筛选数据
4. 表格更新显示查询结果

**代码实现**：
```typescript
const handleSearch = () => {
  const params: any = {};
  if (filters.dateRange) {
    params.startDate = filters.dateRange[0].format('YYYY-MM-DD');
    params.endDate = filters.dateRange[1].format('YYYY-MM-DD');
  }
  if (filters.orgId) params.orgId = filters.orgId;
  if (filters.shiftId) params.shiftId = filters.shiftId;

  setSearchParams(params);
  refetch();
};
```

**API接口**：
```
GET /allocation/line-shifts
参数：startDate, endDate, orgId, shiftId, page, pageSize
返回：PaginatedResponse<LineRecord>
```

### 5.2 新建开线记录

**流程步骤**：
1. 点击"新建"按钮
2. 打开新建对话框，显示表单
3. 用户填写表单字段：
   - 选择组织（必填）
   - 选择开线日期（必填，不能选择未来日期）
   - 选择班次（必填）
   - 选择开始时间（必填）
   - 选择结束时间（必填，必须晚于开始时间）
   - 选择是否参与分摊（默认是）
   - 选择状态（默认"启用"）
4. 点击"确定"按钮
5. 系统验证表单数据
6. 验证通过后，调用API创建记录
7. 创建成功后，关闭对话框，刷新列表
8. 显示成功提示消息

**表单验证规则**：
```typescript
rules: {
  orgId: [{ required: true, message: '请选择组织' }],
  scheduleDate: [{ required: true, message: '请选择开线日期' }],
  shiftId: [{ required: true, message: '请选择班次' }],
  startTime: [{ required: true, message: '请选择开始时间' }],
  endTime: [
    { required: true, message: '请选择结束时间' },
    {
      validator: (_, value) => {
        const startTime = form.getFieldValue('startTime');
        if (startTime && value && value.isBefore(startTime)) {
          return Promise.reject('结束时间必须晚于开始时间');
        }
        return Promise.resolve();
      }
    }
  ],
}
```

**数据转换**：
```typescript
const data: LineRecord = {
  orgId: values.orgId,
  orgName: getOrgNames([values.orgId])[0],
  scheduleDate: scheduleDate,
  startTime: startDateTime,
  endTime: endDateTime,
  shiftId: values.shiftId,
  participateInAllocation: values.participateInAllocation ?? true,
  status: values.status || 'ACTIVE',
};
```

**API接口**：
```
POST /allocation/line-shifts
请求体：LineRecord
返回：创建的LineRecord对象
```

**流程步骤**：
1. 点击表格中某行的"编辑"按钮

5. 系统验证表单数据
6. 验证通过后，调用API更新记录
7. 更新成功后，关闭对话框，刷新列表
8. 显示成功提示消息

**API接口**：

```
PUT /allocation/line-shifts/:id
请求体：LineRecord

```

### 5.4 删除开线记录

### 5.5 复制开线记录

**流程步骤**：
1. 勾选表格中的一条或多条记录

6. 系统为每条选中的记录创建副本，日期更新为目标日期

### 5.6 批量删除开线记录

**流程步骤**：
1. 勾选表格中的一条或多条记录
2. 点击"批量删除"按钮

4. 用户点击"确定"
5. 系统批量调用删除API
6. 删除成功后，清空选择，刷新列表
7. 显示成功提示消息："成功删除X条记录"

**实现方式**：
```typescript
const handleBatchDelete = async () => {

```

## 六、业务规则

### 6.1 日期限制
- 不能选择未来日期作为开线日期
- 结束时间必须晚于开始时间

### 6.2 状态管理
- ACTIVE（启用）：记录有效，参与工时分摊
- INACTIVE（停用）：记录无效，不参与工时分摊

### 6.3 分摊参与
- participateInAllocation = true：该开线记录参与工时分摊计算
- participateInAllocation = false：该开线记录不参与工时分摊计算

### 6.4 数据完整性
- 删除开线记录前，需要检查是否有关联的分摊结果
- 如果有关联数据，应提示用户先删除关联数据

## 七、系统配置依赖

### 7.1 组织数据
- 需要预先配置组织架构
- 组织树形数据来源：`/hr/organizations/tree`

### 7.2 班次数据
- 需要预先配置班次信息
- 班次数据来源：`/hr/shifts`

## 八、权限要求

### 8.1 功能权限
- 查看开线记录：`allocation:line:view`
- 新建开线记录：`allocation:line:create`
- 编辑开线记录：`allocation:line:edit`
- 删除开线记录：`allocation:line:delete`
- 批量操作：`allocation:line:batch`

### 8.2 数据权限
- 用户只能查看和操作自己组织范围内的数据
- 管理员可以查看和操作所有组织的数据

## 九、性能要求

### 9.1 查询性能
- 默认查询最近30天的数据
- 支持分页加载，每页10/20/50/100条可选
- 查询响应时间 < 2秒

### 9.2 批量操作
- 批量复制限制一次最多100条记录
- 批量删除限制一次最多100条记录

## 十、异常处理

### 10.1 网络异常
- API调用失败时，显示错误提示
- 提供重试机制

### 10.2 数据验证异常
- 表单验证失败时，高亮显示错误字段
- 显示具体的错误提示信息

### 10.3 业务逻辑异常
- 删除有关联数据的记录时，提示用户无法删除
- 复制记录时，如果目标日期已存在相同组织+班次的记录，提示用户是否覆盖
