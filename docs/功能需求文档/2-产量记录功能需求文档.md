# 产量记录功能需求文档

## 一、功能概述

产量记录功能用于管理生产线的日常产量数据，包括产品产量信息的录入、编辑、删除和查询。该功能是挣得工时分摊的基础数据来源，记录了每个产品在特定日期、生产线上的实际产量。

## 二、页面路径
- 前端路由：`/allocation/new-production-records`
- 页面组件：`frontend/src/pages/allocation/NewProductionRecordPage.tsx`

## 五、操作流程

### 5.1 日期导航

**流程步骤**：
1. 页面加载时，默认显示今天的数据
2. 用户点击"上一天"按钮：
   - 系统计算当前日期的前一天
   - 更新当前日期状态
   - 重新加载新日期的数据
3. 用户点击"下一天"按钮：
   - 系统检查下一天是否是未来日期
   - 如果是未来日期，显示警告："不能选择未来日期"
   - 如果不是，更新当前日期状态，重新加载数据
4. 用户点击"今天"按钮：
   - 重置当前日期为今天
   - 重新加载今天的数据

**关键代码**：
```typescript
const handleNextDay = () => {
  const nextDay = currentDate.add(1, 'day');
  if (nextDay.isAfter(dayjs(), 'day')) {
    message.warning('不能选择未来日期');
    return;
  }
  setCurrentDate(nextDay);
};

const handlePreviousDay = () => {
  setCurrentDate(currentDate.subtract(1, 'day'));
};

const handleToday = () => {
  setCurrentDate(dayjs());
};
```

### 5.2 查询产量记录

**流程步骤**：
1. 进入页面，系统自动加载今天的产量记录
2. 用户可选择输入筛选条件：
   - 选择账户（支持多选）
   - 选择产品
3. 点击"查询"按钮，系统根据条件筛选数据
4. 表格更新显示查询结果

### 5.3 新建产量记录

**流程步骤**：
1. 点击"新建"按钮
2. 打开新建对话框，显示表单
3. 用户填写表单字段：
   - 选择记录日期（必填，默认为当前查看日期，不能选择未来日期）
   - 选择账户（必填，自动关联组织）
   - 选择产品（必填，自动填充产品名称和编码）
   - 输入实际产量（必填，必须大于0）
   - 系统自动填充标准工时（从产品配置获取）
   - 系统自动计算总标准工时
   - 选择状态（默认"启用"）
4. 点击"确定"按钮
5. 系统验证表单数据
6. 验证通过后，调用API创建记录
7. 创建成功后，关闭对话框，刷新列表
8. 显示成功提示消息

**表单验证规则**：
```typescript
rules: {
  recordDate: [
    { required: true, message: '请选择记录日期' },
    {
      validator: (_, value) => {
        if (value && value.isAfter(dayjs(), 'day')) {
          return Promise.reject('不能选择未来日期');
        }
        return Promise.resolve();
      }
    }
  ],
  accountId: [{ required: true, message: '请选择账户' }],
  productId: [{ required: true, message: '请选择产品' }],
  actualQty: [

    { type: 'number', min: 0.01, message: '产量必须大于0' }
  ],
}
```

**自动计算逻辑**：
```typescript
// 当产品或产量变化时，自动计算总标准工时
useEffect(() => {
  const productId = form.getFieldValue('productId');
  const actualQty = form.getFieldValue('actualQty');

  if (productId && actualQty) {
    // 从产品列表中找到标准工时
    const product = products.find(p => p.id === productId);
    if (product && product.stdHours) {
      const totalStdHours = actualQty * product.stdHours;
      form.setFieldsValue({ totalStdHours });
    }
  }
}, [form.getFieldValue('productId'), form.getFieldValue('actualQty')]);
```

## 六、业务规则

### 6.1 日期限制（重要）
- **不能选择未来日期**：这是核心业务规则
- 新建记录时，记录日期不能晚于今天
- 日期导航时，点击"下一天"会检查是否超过今天
- 如果尝试选择未来日期，显示警告消息："不能选择未来日期"

### 6.2 产量计算
- **总标准工时 = 实际产量 × 单位标准工时**
- 系统自动计算，不允许手动修改
- 实际产量必须大于0
- 标准工时从产品配置中获取

### 6.3 状态管理
- ACTIVE（启用）：记录有效，参与分摊计算
- INACTIVE（停用）：记录无效，不参与分摊计算

### 6.4 数据唯一性
- 同一日期、同一账户、同一产品只能有一条记录
- 如果尝试创建重复记录，系统应提示用户记录已存在

### 6.5 账户与组织关系
- 选择账户后，自动关联到所属组织
- 一个账户只能属于一个组织

## 七、系统配置依赖

### 7.1 产品数据源
- 需要预先配置产品数据源
- 产品数据源配置路径：系统配置 > 数据源配置
- 产品数据必须包含：
  - 产品ID
  - 产品名称
  - 产品编码
  - 标准工时（stdHours）

**获取产品数据**：
```typescript
// 从数据源API获取产品列表
const dataSources = await request.get('/hr/data-sources');
const productDataSource = dataSources.find((ds: any) => ds.code === 'PRODUCT');
const products = await request.get(`/hr/data-sources/${productDataSource.id}/items`);
```

### 7.2 账户数据
- 需要预先配置账户体系
- 账户组织关系必须建立
- 账户数据来源：`/allocation/accounts`

### 7.3 组织架构
- 需要预先配置组织架构
- 组织数据来源：`/hr/organizations`

## 九、性能要求

### 9.1 查询性能
- 日期切换响应时间 < 1秒
- 查询响应时间 < 2秒
- 支持分页加载，每页10/20/50/100条可选

### 9.2 计算性能
- 自动计算总标准工时应实时响应（< 100ms）
- 产品列表加载时间 < 1秒

### 9.3 批量操作
- 批量删除限制一次最多100条记录

## 十、异常处理

### 10.1 日期验证异常
- 尝试选择未来日期时，显示警告："不能选择未来日期"
- 阻止表单提交

### 10.2 网络异常
- API调用失败时，显示错误提示
- 提供重试机制

### 10.3 数据验证异常
- 表单验证失败时，高亮显示错误字段
- 显示具体的错误提示信息
- 产量必须大于0，否则显示："产量必须大于0"

### 10.4 业务逻辑异常
- 创建重复记录时，提示："该日期、账户、产品的产量记录已存在"
- 删除有关联数据的记录时，提示用户无法删除

### 10.5 产品数据异常
- 如果产品数据源未配置，提示："产品数据源未配置，请联系管理员"
- 如果产品的标准工时未配置，提示："产品的标准工时未配置，无法计算总标准工时"

## 十三、与其他功能的关系

### 13.1 与挣得工时分摊的关系
- 产量记录是挣得工时分摊的数据源
- 产品产量作为分摊依据
- 总标准工时用于计算分摊比例

### 13.2 与开线维护的关系
- 开线维护定义了哪些生产线在什么时间段工作
- 产量记录记录了这些生产线的实际产出
- 两者结合使用，完成工时分摊计算
