# 班次属性功能 - 快速配置指南

**版本**: v1.0
**更新日期**: 2026-03-30

## 功能概述

本次更新新增了**班次属性管理**功能，支持：
1. 定义班次属性（如：早班、中班、晚班、开线班次等）
2. 为班次打标签（多选属性）
3. 根据属性过滤班次（在开线维护和产量记录中）

## 快速配置步骤

### 第一步：验证更新

访问系统，确认以下菜单已显示：
- ✓ 排班管理 → 班次属性配置
- ✓ 工时基础配置（新页面，两个页签布局）

### 第二步：配置班次属性

1. 访问：**排班管理 → 班次属性配置**
2. 系统已预置 6 个常用属性：
   - 早班
   - 中班
   - 晚班
   - 夜班
   - 开线班次 ⭐（重要）
   - 特殊班次

3. 可选：新增自定义属性
   - 点击"新增属性"
   - 填写属性代码、名称、描述
   - 选择状态（启用/停用）

### 第三步：为班次打标签

1. 访问：**排班管理 → 班次管理**
2. 编辑或新建班次
3. 在"班次属性"多选框中选择属性
4. 保存

**示例**：
- 早班班次：勾选"早班"和"开线班次"
- 中班班次：勾选"中班"和"开线班次"
- 晚班班次：勾选"晚班"

### 第四步：配置产线开线班次属性

1. 访问：**分摊规则 → 工时基础配置**
2. 找到"产线开线班次属性"配置项
3. 选择"开线班次"属性
4. 保存

### 第五步：验证功能

1. 访问：**分摊规则 → 开线维护**
2. 点击"新增开线记录"
3. 检查班次下拉框
   - ✓ 应该只显示带有"开线班次"属性的班次
   - ✓ 不显示未打标签的班次

4. 访问：**分摊规则 → 产量记录**
5. 点击"新增产量记录"
6. 检查班次下拉框
   - ✓ 应该只显示带有"开线班次"属性的班次

## 常见使用场景

### 场景 1: 只有特定班次可以开线

**需求**：只有早班和中班可以开线，夜班不行

**配置步骤**：
1. 给早班和中班班次打上"开线班次"标签
2. 夜班班次不打此标签
3. 在工时基础配置中选择"开线班次"属性

**效果**：
- 开线维护/产量记录中，下拉框只显示早班和中班

### 场景 2: 不同产线使用不同班次

**需求**：产线A使用早班，产线B使用中班

**配置步骤**：
1. 新增班次属性："产线A"、"产线B"
2. 给对应班次打标签
   - 早班班次：勾选"开线班次"+"产线A"
   - 中班班次：勾选"开线班次"+"产线B"
3. 在工时基础配置中选择"开线班次"+"产线A"+"产线B"

**效果**：
- 开线维护时，如果选择产线A，班次下拉框显示早班
- 开线维护时，如果选择产线B，班次下拉框显示中班

### 场景 3: 特殊班次不需要开线

**需求**：培训班、会议班等特殊班次不需要开线

**配置步骤**：
1. 给正常班次（早、中、晚）打上"开线班次"标签
2. 特殊班次只打"特殊班次"标签，不打"开线班次"

**效果**：
- 开线维护时，不会显示特殊班次

## 配置最佳实践

### 1. 属性命名规范

- 使用简洁的中文名称
- 代码使用拼音或英文缩写
- 避免特殊字符

**示例**：
- 名称："开线班次"，代码："kaixian_banci"
- 名称："产线A"，代码："chanxian_A"

### 2. 属性分类

建议按以下维度定义属性：

| 维度 | 属性示例 |
|------|----------|
| 时段 | 早班、中班、晚班、夜班 |
| 用途 | 开线班次、特殊班次 |
| 产线 | 产线A、产线B、产线C |
| 类型 | 生产班次、维护班次 |

### 3. 排序建议

在属性配置中，通过"排序"字段设置显示顺序：
- 常用属性排在前面（排序号小）
- 不常用属性排在后面（排序号大）

**建议顺序**：
1. 开线班次（10）
2. 时段属性（1-9）
3. 产线属性（20-29）
4. 特殊属性（99+）

## 注意事项

### ⚠️ 重要提示

1. **数据依赖**：
   - 必须先配置班次属性
   - 然后给班次打标签
   - 最后在工时基础配置中选择属性

2. **历史数据**：
   - 本次更新不影响历史班次数据
   - 历史开线记录和产量记录不受影响
   - 只影响新建记录的班次选择

3. **权限控制**：
   - 班次属性配置需要相应权限
   - 建议由管理员统一配置
   - 普通用户只使用班次管理功能

### 💡 使用技巧

1. **快速筛选**：
   - 在班次列表页面，可通过属性列筛选班次
   - 方便查看哪些班次配置了特定属性

2. **批量更新**：
   - 如需批量修改班次属性
   - 可直接在数据库中批量更新 ShiftProperty 表

3. **属性复用**：
   - 一次定义，多处使用
   - 属性可用于其他功能（如排班规则、工资计算等）

## 界面说明

### 班次属性配置页面

**位置**：排班管理 → 班次属性配置

**功能**：
- 新增属性
- 编辑属性
- 删除属性
- 启用/停用属性
- 调整排序

**字段说明**：
- 属性代码：唯一标识，建议英文或拼音
- 属性名称：显示名称，建议中文
- 描述：详细说明
- 状态：启用/停用
- 排序：显示顺序

### 班次管理页面（编辑）

**新增字段**：
- 班次属性：多选下拉框，可选择多个属性

**使用方法**：
1. 点击"新建班次"或编辑现有班次
2. 填写班次基本信息
3. 在"班次属性"下拉框中选择属性
4. 可多选
5. 保存

### 工时基础配置页面

**位置**：分摊规则 → 工时基础配置

**页签布局**：
- 页签1：工时基础配置
- 页签2：产品配置

**重要配置项**：
- 产线对应层级
- 产线开线班次属性 ⭐
- 间接分摊配置

**配置产线开线班次属性**：
1. 进入"工时基础配置"页签
2. 找到"产线开线班次属性"配置项
3. 点击下拉框，选择属性（可多选）
4. 下方会显示"匹配的班次数量"
5. 点击"保存配置"

## 故障排查

### 问题 1：班次下拉框为空

**原因**：
- 未在工时基础配置中选择属性
- 或班次未配置对应属性

**解决**：
1. 检查工时基础配置是否已保存
2. 检查班次是否已打上相应标签

### 问题 2：看不到"班次属性"字段

**原因**：
- 前端未更新
- 或权限不足

**解决**：
1. 刷新浏览器页面（Ctrl+F5）
2. 清除浏览器缓存
3. 重新登录

### 问题 3：保存后属性未生效

**原因**：
- 网络延迟
- 或缓存未刷新

**解决**：
1. 等待 1-2 秒后重新查看
2. 刷新页面
3. 检查后端日志
