# 人员信息标准字段模版

本文档定义了人事管理系统中人员信息的标准字段模版，用于快速构建完整的人事档案。

---

## 一、基本信息页签 (basic_info)

### 1.1 个人信息

| 字段编码 | 字段名称 | 字段类型 | 是否必填 | 说明 |
|---------|---------|---------|---------|------|
| employee_no | 工号 | TEXT | 是 | 员工唯一编号，系统自动生成或手动输入 |
| name | 姓名 | TEXT | 是 | 员工真实姓名 |
| gender | 性别 | SELECT_SINGLE | 是 | 男/女 |
| id_card | 身份证号 | TEXT | 是 | 18位身份证号码 |
| birth_date | 出生日期 | DATE | 是 | 出生日期 |
| age | 年龄 | NUMBER | 否 | 自动计算 |
| nation | 民族 | SELECT_SINGLE | 否 | 汉族/回族等 |
| marital_status | 婚姻状况 | SELECT_SINGLE | 是 | 未婚/已婚/离异/丧偶 |
| political_status | 政治面貌 | SELECT_SINGLE | 否 | 党员/团员/群众/民主党派 |
| native_place | 籍贯 | TEXT | 否 | 祖籍地址 |
| household_register | 户口所在地 | TEXT | 否 | 户口地址 |
| current_address | 现居住地址 | TEXT | 否 | 当前居住地址 |
| photo | 照片 | IMAGE | 否 | 员工证件照 |

## 字段类型说明

| 类型代码 | 类型名称 | 数据格式 | 说明 |

## 页签配置建议

### 默认页签顺序

1. 基本信息 (basic_info) - 必填项最多，优先级最高
2. 联系信息 (contact_info) - 基础联系信息
3. 工作信息 (work_info) - 核心工作相关信息
4. 学历信息 (education_info) - 教育背景
5. 合同信息 (contract_info) - 劳动合同
6. 薪资信息 (salary_info) - 薪资社保
7. 考勤信息 (attendance_info) - 考勤打卡
8. 技能证书 (skills_certificates) - 职业技能
9. 工作经历 (work_experience) - 历史履历
10. 家庭信息 (family_info) - 家庭成员
11. 银行卡信息 (bank_info) - 薪资发放
12. 其他信息 (other_info) - 补充信息

### 可选页签

根据企业实际需求，可选页签包括：
- 培训记录
- 奖惩记录
- 调岗记录
- 考核记录
- 资产装备
- 系统权限

## 实施建议

### 1. 分阶段实施

**第一阶段（必需字段）：**
- 基本信息页签：工号、姓名、性别、身份证、手机号、组织、职位、入职日期
- 工作信息页签：部门、职级、员工类型、状态

**第二阶段（核心字段）：**
- 联系信息页签：邮箱、紧急联系人
- 学历信息页签：最高学历
- 合同信息页签：合同类型、合同期限

**第三阶段（扩展字段）：**
- 薪资信息、考勤信息
- 工作经历、技能证书
- 家庭信息等

### 2. 系统初始化脚本

建议提供系统初始化脚本，在系统部署时自动创建以上标准页签和字段。

### 3. 字段校验规则

- 身份证号：18位校验
- 手机号：11位手机号校验
- 邮箱：邮箱格式校验
- 工号：唯一性校验
- 日期字段：开始日期不能大于结束日期

### 4. 权限控制

- 系统内置字段不允许删除
- 系统内置页签不允许删除
- 敏感字段（薪资、身份证等）需要特殊权限查看
- 字段显示权限可配置
