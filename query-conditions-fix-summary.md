# 查询条件显示问题修复总结

## 问题描述
排班管理页面（SchedulePage.tsx）、打卡记录页面（PunchRecordPage.tsx）、工时明细管理页面（WorkHourDetailPage.tsx）配置了查询条件，但在页面未显示出来。

## 根本原因
数据库中只有 `work-hour-details` 页面的查询条件配置，缺少以下两个页面的配置：
1. `schedule-management` - 排班管理页面
2. `punch-records` - 打卡记录页面

此外，还缺少以下数据源：
- `PUNCH_TYPE` - 打卡类型
- `PUNCH_SOURCE` - 打卡来源

## 解决方案

### 1. 为排班管理页面添加查询条件配置
- **工号** (`employeeNo`) - 文本类型
- **姓名** (`name`) - 文本类型
- **所属组织** (`orgId`) - 组织类型

### 2. 为打卡记录页面添加查询条件配置
- **工号** (`employeeNo`) - 文本类型
- **姓名** (`name`) - 文本类型
- **打卡类型** (`punchType`) - 下拉选择类型（关联 PUNCH_TYPE 数据源）
- **来源** (`source`) - 下拉选择类型（关联 PUNCH_SOURCE 数据源）

### 3. 创建缺失的数据源
- **PUNCH_TYPE** (打卡类型)
  - 签入 (IN)
  - 签出 (OUT)

- **PUNCH_SOURCE** (打卡来源)
  - 自动 (AUTO)
  - 手动 (MANUAL)

## 验证结果
所有三个页面现在都有完整的查询条件配置：

### 排班管理页面 (schedule-management)
```
✓ 工号 (文本输入)
✓ 姓名 (文本输入)
✓ 所属组织 (组织选择器)
```

### 打卡记录页面 (punch-records)
```
✓ 工号 (文本输入)
✓ 姓名 (文本输入)
✓ 打卡类型 (下拉选择：签入/签出)
✓ 来源 (下拉选择：自动/手动)
```

### 工时明细管理页面 (work-hour-details)
```
✓ 工号 (文本输入)
✓ 姓名 (文本输入)
✓ 性别 (下拉选择)
✓ 所属组织 (组织选择器)
✓ 员工类型 (下拉选择)
✓ 入职日期 (日期选择)
```

## 数据库变更
执行了以下SQL脚本：
1. `add-query-condition-configs.sql` - 添加查询条件配置
2. `add-missing-datasources.sql` - 创建缺失的数据源

所有配置都已验证成功，查询条件应该可以在页面上正常显示了。

## 使用说明
刷新页面后，您应该能看到以下查询条件：

### 排班管理页面
- 可以通过工号、姓名、所属组织进行筛选
- 配合日期范围选择器使用

### 打卡记录页面
