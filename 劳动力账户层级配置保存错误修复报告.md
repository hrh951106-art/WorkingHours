# 劳动力账户层级配置保存错误修复报告

## 问题描述
新增劳动力账户层级配置保存时提示服务器内部错误

## 问题诊断

### 1. 数据库表结构检查

**AccountHierarchyLevelDetail** 表结构：
```sql
CREATE TABLE AccountHierarchyLevelDetail (
  id INTEGER PRIMARY KEY,
  configId INTEGER NOT NULL,        -- 外键，指向层级配置
  level INTEGER NOT NULL,            -- 层级序号
  levelName TEXT NOT NULL,           -- 层级名称
  levelCode TEXT,                    -- 层级代码
  parentLevelId INTEGER,             -- 父层级ID
  parentLevelName TEXT,              -- 父层级名称
  accountCodePrefix TEXT,            -- 账户代码前缀
  status TEXT DEFAULT 'ACTIVE',      -- 状态
  createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
  updatedAt DATETIME,
  FOREIGN KEY (configId) REFERENCES AccountHierarchyConfig(id)
);
```

### 2. 代码问题定位

在 `backend/src/modules/account/account.service.ts` 的 `fetchHierarchyLevelDetails` 方法中，返回的数据结构使用了错误的字段名：

**错误的字段映射**：
```typescript
{
  hierarchyConfigId,  // ❌ 应该是 configId
  optionCode,         // ❌ 应该是 levelCode
  optionId,           // ❌ 表中不存在此字段
  optionLabel,        // ❌ 应该是 levelName
  sortOrder,          // ❌ 表中不存在此字段
  status: 'ACTIVE',
}
```

**正确的字段映射**：
```typescript
{
  configId: hierarchyConfigId,  // ✅ 正确的外键字段
  level: index + 1,              // ✅ 层级序号
  levelCode: org.code,          // ✅ 层级代码
  levelName: org.name,          // ✅ 层级名称
  status: 'ACTIVE',             // ✅ 状态
}
```

### 3. 其他相关问题

1. **syncHierarchyLevelDetails** 方法中的删除条件使用了错误的字段名：
   - ❌ `where: { hierarchyConfigId: id }`
   - ✅ `where: { configId: id }`

2. **getHierarchyLevelsWithDetails** 方法中的关联字段名和排序字段错误：
   - ❌ `levelDetails` (应该是 `details`)
   - ❌ `orderBy: { sortOrder: 'asc' }` (表中没有sortOrder字段)
   - ✅ `details`
   - ✅ `orderBy: { level: 'asc' }`

## 修复方案

### 修改1: fetchHierarchyLevelDetails 方法 - 组织类型映射
**文件**: `backend/src/modules/account/account.service.ts:165-172`

```typescript
// 修改前
return organizations.map((org: any, index: number) => ({
  hierarchyConfigId,
  optionCode: org.code,
  optionId: String(org.id),
  optionLabel: org.name,
  sortOrder: index,
  status: 'ACTIVE',
}));

// 修改后
return organizations.map((org: any, index: number) => ({
  configId: hierarchyConfigId,
  level: index + 1,
  levelCode: org.code,
  levelName: org.name,
  status: 'ACTIVE',
}));
```

### 修改2: fetchHierarchyLevelDetails 方法 - 自定义字段映射
**文件**: `backend/src/modules/account/account.service.ts:227-234`

```typescript
// 修改前
return dataSource.options.map((option: any) => ({
  hierarchyConfigId,
  optionCode: option.value,
  optionId: String(option.id),
  optionLabel: option.label,
  sortOrder: option.sort,
  status: 'ACTIVE',
}));

// 修改后
return dataSource.options.map((option: any, index: number) => ({
  configId: hierarchyConfigId,
  level: index + 1,
  levelCode: option.value,
  levelName: option.label,
  status: 'ACTIVE',
}));
```

### 修改3: syncHierarchyLevelDetails 方法
**文件**: `backend/src/modules/account/account.service.ts:118-120`

```typescript
// 修改前
await tx.accountHierarchyLevelDetail.deleteMany({
  where: { hierarchyConfigId: id },
});

// 修改后
await tx.accountHierarchyLevelDetail.deleteMany({
  where: { configId: id },
});
```

### 修改4: getHierarchyLevelsWithDetails 方法
**文件**: `backend/src/modules/account/account.service.ts:383-386`

```typescript
// 修改前
include: {
  levelDetails: {
    where: { status: 'ACTIVE' },
    orderBy: { sortOrder: 'asc' },
  },
},

// 修改后
include: {
  details: {
    where: { status: 'ACTIVE' },
    orderBy: { level: 'asc' },
  },
},
```
