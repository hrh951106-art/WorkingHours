# 金额政策层级匹配功能实现说明

## 📋 功能概述

实现了金额政策的账户路径层级匹配功能，支持按账户层级进行灵活的匹配，而不是仅限于精确匹配或前缀匹配。

## 🎯 问题背景

用户反馈金额政策配置的账户路径（如 "///大桶/焊接//"）无法匹配到实际账户路径（如 "大华富阳工厂/W1总装车间/W1总装车间L2产线/大桶/焊接/-/-"）。

需求是：
- 只需要匹配对应层级的值
- 例如：线体工时 ///大桶/焊接// 应该能匹配到所有产品层级为大桶、工序层级为焊接的工时数据
- 其他层级（如工厂、车间、产线）可以为空（表示任意值）

## ✅ 解决方案

### 1. 新增 LEVEL 匹配模式

在原有的 EXACT（精确匹配）和 PREFIX（前缀匹配）基础上，新增了 LEVEL（层级匹配）模式：

**层级匹配规则：**
- 账户路径按 "/" 分割成多个层级
- 空层级（// 或 -）表示任意值，不参与匹配
- 非空层级必须精确匹配对应位置的值
- 所有非空层级都匹配才算成功

**示例：**
```
政策路径：///大桶/焊接//
可以匹配：
✅ 大华富阳工厂/W1总装车间/W1总装车间L2产线/大桶/焊接/-/-
✅ DH/DH01/DH01001/WELDING/大桶/焊接/Any/Value
❌ 大华富阳工厂/W1总装车间/W1总装车间L2产线/小桶/焊接/-/-  (产品不匹配)
❌ 大华富阳工厂/W1总装车间/W1总装车间L2产线/大桶/包装/-/-  (工序不匹配)
```

### 2. 后端实现

**文件：** `backend/src/modules/amount/amount-policy.service.ts`

新增了 `matchAccountPathByLevel` 私有方法：

```typescript
private matchAccountPathByLevel(policyPath: string, actualPath: string): boolean {
  // 分割路径，保留层级位置
  const policyParts = policyPath.split('/');
  const actualParts = actualPath.split('/');

  // 如果政策路径为空或只有"/"，则匹配所有
  const hasValidPolicyValue = policyParts.some(p => p && p !== '-');
  if (!hasValidPolicyValue) {
    return true;
  }

  // 逐层检查
  let matchCount = 0;
  let requiredMatches = 0;
  const maxLevel = Math.max(policyParts.length, actualParts.length);

  for (let i = 0; i < maxLevel; i++) {
    const policyValue = policyParts[i] || '';
    const actualValue = actualParts[i] || '';

    // 如果政策中该层级为空、空字符串或"-"，则跳过（表示任意值）
    if (!policyValue || policyValue === '-' || policyValue === '') {
      continue;
    }

    requiredMatches++;

    // 检查实际路径的该层级是否匹配
    if (actualValue === policyValue) {
      matchCount++;
    } else {
      // 任何一个必需层级不匹配就失败
      return false;
    }
  }

  // 所有必需层级都匹配才算成功
  return matchCount === requiredMatches && requiredMatches > 0;
}
```

更新了 `matchPolicy` 方法，支持 3 种匹配模式：
- LEVEL: 层级匹配
- EXACT: 精确匹配
- PREFIX: 前缀匹配

## 🎯 使用说明

### 前端操作

1. **创建/编辑金额政策**
   - 进入：计算管理 → 金额政策页面
   - 点击"新建规则"或编辑已有规则
   - 选择"账户匹配模式"（默认为"层级匹配"）

2. **选择匹配模式**
   - **层级匹配（推荐）**：适合按产品、工序等维度批量匹配
     - 例如：///大桶/焊接// 匹配所有大桶+焊接的组合
   - **精确匹配**：适合匹配特定账户
     - 例如：DH/DH01/DH01001/WELDING 只匹配这一个账户
   - **前缀匹配**：适合匹配某个账户下的所有子账户
     - 例如：DH/DH01 匹配所有以 DH/DH01 开头的账户

3. **配置账户路径**
   - 使用账户选择器选择账户
   - 系统自动转换为层级格式
   - 可以手动编辑路径（使用 / 分隔层级）

### 配置示例

**场景 1：所有焊接工时享受 1.5 倍金额**
```
规则名称：焊接工时1.5倍
规则类型：翻倍
倍数：1.5
账户路径：////焊接//  (只指定工序为焊接)
匹配模式：层级匹配
出勤代码：A02
```

**场景 2：大桶产品所有工序享受 1.2 倍金额**
```
规则名称：大桶产品1.2倍
规则类型：翻倍
倍数：1.2
账户路径：///大桶///  (只指定产品为大桶)
匹配模式：层级匹配
出勤代码：A02
```

**场景 3：特定产线的特定工序享受 2 倍金额**
```
规则名称：W1L2产线焊接2倍
规则类型：翻倍
倍数：2.0
账户路径：/W1总装车间/W1总装车间L2产线/大桶/焊接//  (精确到产线、产品、工序)
匹配模式：层级匹配
出勤代码：A02
```

## 🔧 技术细节

### 账户层级结构

根据系统设计，账户路径包含 7 个层级：

| 层级 | 名称 | 说明 | 示例值 |
|-----|------|------|--------|
| 1 | 工厂 | 顶层组织 | 大华富阳工厂 |
| 2 | 车间 | 车间层级 | W1总装车间 |
| 3 | 产线 | 产线层级 | W1总装车间L2产线 |
| 4 | 产品 | 产品类别 | 大桶 |
| 5 | 工序 | 工序类型 | 焊接、包装 |
| 6 | Level 6 | 预留层级 | - |
| 7 | Level 7 | 预留层级 | - |

### 匹配性能

- 查询阶段：获取所有激活的金额政策（不在数据库层面过滤）
- 匹配阶段：在 JavaScript 中逐个匹配政策
- 优先级排序：按 priority DESC, effectiveDate DESC 排序
- 返回结果：返回第一个匹配的政策

### 兼容性

- 完全兼容现有的 EXACT 和 PREFIX 匹配模式
- 现有政策不会自动转换为 LEVEL 模式
- 新建政策默认使用 LEVEL 模式
- 可以手动将现有政策编辑为 LEVEL 模式

## 📝 注意事项

1. **路径格式**
   - 必须使用 "/" 作为分隔符
   - 空层级使用 "//" 或 "-"
   - 建议保持 7 个层级（不足的用 "/" 或 "-" 填充）

2. **配置建议**
   - 优先使用层级匹配，灵活性最高
   - 尽量指定必要的层级，避免误匹配
   - 例如：如果只需要区分工序，就只指定第 5 层级

3. **测试验证**
   - 配置后建议先测试匹配结果
   - 可以通过计算结果页面验证金额是否正确
   - 使用重新计算功能更新已有数据

4. **性能考虑**
   - 当前实现在内存中进行匹配
   - 如果政策数量很大（>1000），可能需要优化查询
   - 建议定期清理过期或不再使用的政策

## 🎉 总结

通过实现层级匹配功能，金额政策的配置更加灵活和强大：

✅ 支持按任意层级组合进行匹配
✅ 简化了配置流程（不需要精确配置完整路径）
✅ 提高了配置的复用性（一个政策可匹配多个账户）
✅ 保持了向后兼容性（不影响现有配置）

现在可以使用 "///大桶/焊接//" 这样的路径来匹配所有符合条件账户的工时数据了！
