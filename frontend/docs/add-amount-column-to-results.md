## 🔑 金额计算逻辑

金额根据以下公式计算：

1. **无金额规则**：`金额 = 人员系数 × 工时数`
2. **有金额规则（MULTIPLY）**：`金额 = 人员系数 × 倍数 × 工时数`
3. **有金额规则（ADD）**：`金额 = 人员系数 + 固定值 × 工时数`
4. **有金额规则（CUSTOM）**：`金额 = 固定值 × 工时数`

### 示例计算
```
人员系数：20
金额规则：A01出勤代码 + WELDING账户 + 1.5倍

A01（3小时）：3 × 20 × 1.5 = ¥90.00
A02（4小时）：4 × 20 × 1.0 = ¥80.00（无匹配规则）
```

## 📱 用户体验

### 可视化优势
1. **一目了然**：金额以橙色高亮显示，与工时的绿色形成对比
2. **清晰易读**：统一格式（¥符号 + 2位小数）
3. **信息完整**：同时展示工时和金额，便于核对
4. **空值友好**：无金额时显示"-"，不会造成困惑

### 数据准确性
- 金额数据来自后端CalcResult和WorkHourResult表的amount字段
- 数据在计算工时时自动计算，无需手动输入
- 支持多账户分别计算并累加

## 🔗 关联功能

金额计算涉及以下功能模块：
1. **员工系数管理**：在员工管理中配置金额系数
2. **金额政策管理**：在金额政策中配置规则
3. **计算出勤代码配置**：启用出勤代码的金额计算
4. **考勤规则组配置**：关联金额政策到规则组

## 📝 使用指南

### 查看金额
1. 进入"系统管理" → "计算管理" → "计算结果"
2. 选择"考勤工时结果"或"精益工时结果"页签
3. 在表格中查看"金额"列
4. 金额会根据人员系数和金额规则自动计算

### 确保金额计算
要确保金额正确计算，需要：
1. ✅ 配置员工系数（EmployeeCoefficient表）
2. ✅ 配置金额政策（AmountPolicy表）
3. ✅ 启用出勤代码金额计算（CalculationAttendanceCode.calculateAmount = true）
4. ✅ 关联金额政策到考勤规则组（AttendanceRuleGroupDetail.amountPolicyIds）
5. ✅ 运行工时计算
